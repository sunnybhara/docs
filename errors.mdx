Every NAWA API error returns a consistent JSON envelope with machine-readable codes and human-friendly guidance.

## Error response format

```json
{
  "type": "error",
  "error": {
    "type": "invalid_request_error",
    "code": "missing_required_param",
    "message": "The 'text' parameter is required.",
    "display_message": "Please provide the comment text to classify.",
    "param": "text",
    "doc_url": "https://developers.trynawa.com/errors#missing_required_param",
    "suggested_action": "Include the 'text' field in your request body."
  },
  "request_id": "req_nw_4c11w71e44cb"
}
```

### Error object fields

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Always `"error"` at the top level |
| `error.type` | string | Error category (see taxonomy below) |
| `error.code` | string | Machine-readable error code |
| `error.message` | string | Technical description for developers |
| `error.display_message` | string | User-safe message suitable for end users |
| `error.param` | string or null | The parameter that caused the error, if applicable |
| `error.doc_url` | string | Link to documentation for this specific error |
| `error.suggested_action` | string | What to do to fix the error |
| `request_id` | string | Unique request identifier (format: `req_nw_xxx`) |

<Tip>
  Always include the `request_id` from error responses when contacting support. It helps us trace exactly what happened.
</Tip>

## Error taxonomy

### `invalid_request_error` (400)

The request was malformed or missing required parameters.

<AccordionGroup>
  <Accordion title="missing_required_param">
    **Cause:** A required parameter is missing from the request body.

    **Common triggers:**
    - Missing `text` field in `/v1/classify`
    - Missing `rubric` field in `/v1/rubric-classify`

    **Fix:** Check the API reference for required parameters and include them all.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "missing_required_param",
        "message": "The 'text' parameter is required.",
        "param": "text",
        "doc_url": "https://developers.trynawa.com/errors#missing_required_param",
        "suggested_action": "Include the 'text' field in your request body."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="invalid_param_value">
    **Cause:** A parameter has an invalid value or type.

    **Common triggers:**
    - `strictness` not one of: `low`, `medium`, `high`
    - `text` is empty or not a string
    - `rubric.categories` exceeds maximum of 20

    **Fix:** Verify parameter values match the expected types and allowed values.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "invalid_param_value",
        "message": "Invalid value for 'strictness'. Must be one of: low, medium, high.",
        "param": "strictness",
        "doc_url": "https://developers.trynawa.com/errors#invalid_param_value",
        "suggested_action": "Use a valid strictness value."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="text_too_long">
    **Cause:** The `text` parameter exceeds the maximum allowed length of 5,000 characters.

    **Fix:** Truncate or split long texts before sending.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "text_too_long",
        "message": "Text exceeds maximum length of 5000 characters (received 7234).",
        "param": "text",
        "doc_url": "https://developers.trynawa.com/errors#text_too_long",
        "suggested_action": "Shorten the text to 5000 characters or fewer."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="invalid_json">
    **Cause:** The request body is not valid JSON.

    **Fix:** Validate your JSON before sending. Check for trailing commas, unquoted keys, or encoding issues.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "invalid_json",
        "message": "Request body is not valid JSON.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors#invalid_json",
        "suggested_action": "Validate your JSON payload before sending."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="method_not_allowed">
    **Cause:** The HTTP method is not supported for this endpoint.

    **Fix:** Check the API reference for the correct method (e.g., POST for classify, GET for health).

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "method_not_allowed",
        "message": "Method GET is not allowed. Use POST.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors#method_not_allowed",
        "suggested_action": "Use the correct HTTP method for this endpoint."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>
</AccordionGroup>

### `authentication_error` (401)

The API key is missing, invalid, expired, or revoked.

<AccordionGroup>
  <Accordion title="missing_auth_header">
    **Cause:** No `Authorization` header was provided.

    **Fix:** Include the header: `Authorization: Bearer nawa_live_sk_xxx`

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "missing_auth_header",
        "message": "No API key provided in the Authorization header.",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/api/authentication",
        "suggested_action": "Add 'Authorization: Bearer YOUR_API_KEY' to your request headers."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="invalid_auth_format">
    **Cause:** The Authorization header value is not in the expected `Bearer nawa_...` format.

    **Fix:** Use the format `Authorization: Bearer nawa_live_sk_xxx` or `Authorization: Bearer nawa_test_sk_xxx`.

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "invalid_auth_format",
        "message": "Authorization header must use 'Bearer nawa_...' format.",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/api/authentication",
        "suggested_action": "Use the format: Authorization: Bearer YOUR_API_KEY"
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="invalid_api_key">
    **Cause:** The API key is not recognized. It may be malformed or deleted.

    **Fix:** Verify the key is correct. Create a new key from the dashboard if needed.

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "invalid_api_key",
        "message": "The API key provided is not valid.",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/api/authentication",
        "suggested_action": "Check that your API key is correct. Create a new key from the dashboard if needed."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="key_expired">
    **Cause:** The free API key has exceeded its 14-day expiry.

    **Fix:** Create a new free key from the dashboard. Live keys do not expire.

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "key_expired",
        "message": "This free API key has expired (14-day limit).",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/api/authentication",
        "suggested_action": "Create a new free key from the dashboard, or purchase credits for a live key."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="key_revoked">
    **Cause:** The API key has been manually revoked.

    **Fix:** Use a different active key or create a new one.

    ```json
    {
      "type": "error",
      "error": {
        "type": "permission_error",
        "code": "key_revoked",
        "message": "This API key has been revoked.",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/api/authentication",
        "suggested_action": "Create a new API key from the dashboard."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>
</AccordionGroup>

### `insufficient_credits` (402)

Your account balance is insufficient for the requested operation.

<AccordionGroup>
  <Accordion title="balance_exhausted">
    **Cause:** Your credit balance has reached $0 and the grace period has ended. The grace period allows up to 1 hour or $2.00 of additional usage after your balance hits zero.

    **Fix:** Purchase a credit pack from the dashboard to restore access instantly.

    ```json
    {
      "type": "error",
      "error": {
        "type": "insufficient_credits",
        "code": "balance_exhausted",
        "message": "Credit balance exhausted. Purchase credits to continue.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/billing",
        "suggested_action": "Buy a credit pack at trynawa.com/developers/keys."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>
</AccordionGroup>

### `rate_limit_error` (429)

You have exceeded the rate limit for your current tier.

<AccordionGroup>
  <Accordion title="rate_limit_exceeded">
    **Cause:** Too many requests in the current time window.

    **Fix:** Wait until the `X-RateLimit-Reset` time, then retry. Consider upgrading your tier.

    ```json
    {
      "type": "error",
      "error": {
        "type": "rate_limit_error",
        "code": "rate_limit_exceeded",
        "message": "Rate limit exceeded: 120 requests per minute for Growth tier.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/rate-limits",
        "suggested_action": "Wait until the rate limit resets or upgrade your tier."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```

    The `Retry-After` response header tells you how many seconds to wait. The `X-NAWA-RateLimit-Reason` header provides additional context.
  </Accordion>

  <Accordion title="sandbox_exhausted">
    **Cause:** You have used all 100 lifetime requests on your free key.

    **Fix:** Buy credits to unlock a live key.

    ```json
    {
      "type": "error",
      "error": {
        "type": "rate_limit_error",
        "code": "sandbox_exhausted",
        "message": "Free key lifetime limit of 100 requests reached.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/billing",
        "suggested_action": "Buy credits to create a live API key at trynawa.com/developers/keys."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>
</AccordionGroup>

### `api_error` (500)

An unexpected error occurred on the NAWA side.

<AccordionGroup>
  <Accordion title="internal_error">
    **Cause:** An unexpected internal failure.

    **Fix:** Retry the request. If the issue persists, contact support with the `request_id`.

    ```json
    {
      "type": "error",
      "error": {
        "type": "api_error",
        "code": "internal_error",
        "message": "An unexpected error occurred. Please retry.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors",
        "suggested_action": "Retry the request. Contact support if the issue persists."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>

  <Accordion title="provider_failure">
    **Cause:** The upstream AI provider experienced a failure. NAWA automatically falls back to a secondary provider when available.

    **Fix:** Retry after a brief delay. Check [status.trynawa.com](https://status.trynawa.com) for incidents.

    ```json
    {
      "type": "error",
      "error": {
        "type": "api_error",
        "code": "provider_failure",
        "message": "AI provider temporarily unavailable.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors",
        "suggested_action": "Retry after a few seconds. Check status.trynawa.com for updates."
      },
      "request_id": "req_nw_abc123def456"
    }
    ```
  </Accordion>
</AccordionGroup>

## Handling errors in code

<CodeGroup>

```bash cURL
# Check for errors in the response
response=$(curl -s -w "\n%{http_code}" \
  -X POST https://api.trynawa.com/v1/classify \
  -H "Authorization: Bearer $NAWA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text": "test"}')

http_code=$(echo "$response" | tail -1)
body=$(echo "$response" | sed '$d')

if [ "$http_code" != "200" ]; then
  echo "Error ($http_code): $body"
fi
```

```typescript TypeScript
import { Nawa, NawaAuthError, NawaRateLimitError } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.classify({
  text: 'test comment'
})

if (error) {
  switch (error.type) {
    case 'authentication_error':
      console.error('Check your API key:', error.message)
      break
    case 'rate_limit_error':
      console.log('Rate limited, retrying after:', error.retryAfter)
      break
    case 'insufficient_credits':
      console.error('Buy credits:', error.suggested_action)
      break
    default:
      console.error('API error:', error.message)
  }
}
```

```python Python
from nawa import Nawa
from nawa.errors import NawaAuthError, NawaRateLimitError

nawa = Nawa(api_key="your_api_key")

result = nawa.classify(text="test comment")

if result.error:
    if result.error.type == "authentication_error":
        print(f"Check your API key: {result.error.message}")
    elif result.error.type == "rate_limit_error":
        print(f"Rate limited, retry after: {result.error.retry_after}")
    elif result.error.type == "insufficient_credits":
        print(f"Buy credits: {result.error.suggested_action}")
    else:
        print(f"API error: {result.error.message}")
```
