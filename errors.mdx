Every NAWA API error returns a consistent JSON envelope with machine-readable codes and human-friendly guidance.

## Error response format

```json
{
  "type": "error",
  "error": {
    "type": "invalid_request_error",
    "code": "missing_field",
    "message": "`text` is required",
    "display_message": "The required field `text` is missing.",
    "param": "text",
    "doc_url": "https://developers.trynawa.com/errors#missing_field",
    "suggested_action": "Include `text` in your request body."
  },
  "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
}
```

### Error object fields

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Always `error` at the top level |
| `error.type` | string | Error category (see taxonomy below) |
| `error.code` | string | Machine-readable error code |
| `error.message` | string | Technical description for developers |
| `error.display_message` | string | User-safe message suitable for end users |
| `error.param` | string \| null | The parameter that caused the error, if applicable |
| `error.doc_url` | string | Link to documentation for this specific error |
| `error.suggested_action` | string | What to do to fix the error |
| `request_id` | string | Unique request ID for debugging |

<Tip>
  Always include the `request_id` from error responses when contacting support. It helps us trace exactly what happened.
</Tip>

## Error taxonomy

### `invalid_request_error` (400)

The request was malformed or missing required parameters.

<AccordionGroup>
  <Accordion title="missing_field">
    **Cause:** A required parameter is missing from the request body.

    **Common triggers:**
    - Missing `text` field in `/v1/classify`
    - Missing `rubric` in `/v1/rubric-classify`
    - Missing `classification_id` or `rating` in `/v1/feedback`

    **Fix:** Check the API reference for required parameters and include them all.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "missing_field",
        "message": "`text` is required",
        "display_message": "The required field `text` is missing.",
        "param": "text",
        "doc_url": "https://developers.trynawa.com/errors#missing_field",
        "suggested_action": "Include `text` in your request body."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>

  <Accordion title="invalid_field">
    **Cause:** A parameter has an invalid value or type.

    **Common triggers:**
    - `provider` not one of: `claude`, `gemini`, `allam`
    - `tone` not one of: `friendly`, `professional`, `casual`, `formal`
    - `text` is empty or not a string
    - `rubric.categories` has more than 20 items

    **Fix:** Verify parameter values match the expected types and allowed values.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "invalid_field",
        "message": "Invalid value for `provider`. Must be one of: claude, gemini, allam.",
        "display_message": "The provider value is not valid.",
        "param": "provider",
        "doc_url": "https://developers.trynawa.com/errors#invalid_field",
        "suggested_action": "Use a valid provider value."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>

  <Accordion title="invalid_json">
    **Cause:** The request body is not valid JSON.

    **Fix:** Ensure your request body is well-formed JSON with proper quoting and escaping.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "invalid_json",
        "message": "Request body is not valid JSON.",
        "display_message": "We couldn't parse the request body.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors#invalid_json",
        "suggested_action": "Ensure your request body is valid JSON."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>

  <Accordion title="method_not_allowed">
    **Cause:** The HTTP method is not supported for this endpoint.

    **Fix:** Use the correct HTTP method listed in the API reference.

    ```json
    {
      "type": "error",
      "error": {
        "type": "invalid_request_error",
        "code": "method_not_allowed",
        "message": "Method GET not allowed. Use POST.",
        "display_message": "This endpoint only accepts POST requests.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors#method_not_allowed",
        "suggested_action": "Use the POST method for this endpoint."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>
</AccordionGroup>

### `authentication_error` (401)

The API key is missing, invalid, expired, or revoked.

<AccordionGroup>
  <Accordion title="invalid_api_key">
    **Cause:** The API key is not recognized, expired, or revoked.

    **Fix:** Verify the key is correct, active, and matches the environment (free/live).

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "invalid_api_key",
        "message": "API key not found",
        "display_message": "The provided API key is not valid.",
        "param": "Authorization",
        "doc_url": "https://developers.trynawa.com/authentication",
        "suggested_action": "Check your API key in the NAWA dashboard."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>

  <Accordion title="sandbox_exhausted">
    **Cause:** You have used all 100 lifetime requests on your free key.

    **Fix:** Purchase credits to unlock a live key.

    ```json
    {
      "type": "error",
      "error": {
        "type": "authentication_error",
        "code": "sandbox_exhausted",
        "message": "Free key lifetime limit of 100 requests reached.",
        "display_message": "Your free key has reached its request limit.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/billing",
        "suggested_action": "Buy credits to create a live API key at trynawa.com/developers/keys."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>
</AccordionGroup>

<AccordionGroup>

    ```json
        "type": "permission_error",

        "display_message": "Your account has insufficient credits.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/billing",
        "suggested_action": "Purchase credits at https://trynawa.com/billing"
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>
</AccordionGroup>

### `rate_limit_error` (429)

You have exceeded the rate limit for your current tier.

<AccordionGroup>
  <Accordion title="rate_limit_exceeded">
    **Cause:** Too many requests in the current time window.

    **Fix:** Wait for the `Retry-After` period, then retry. Consider upgrading your tier for higher limits.

    ```json
    {
      "type": "error",
      "error": {
        "type": "rate_limit_error",
        "code": "rate_limit_exceeded",
        "message": "Rate limit exceeded: minute_limit",
        "display_message": "You have exceeded the request rate limit for your plan.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/rate-limits",
        "suggested_action": "Wait 45s and retry, or upgrade your plan for higher limits."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```

    Additional headers on 429 responses:

    | Header | Description |
    |--------|-------------|
    | `Retry-After` | Seconds until the rate limit resets |
    | `X-NAWA-RateLimit-Reason` | Which limit was hit (e.g., `minute_limit`) |
  </Accordion>
</AccordionGroup>

### `not_found_error` (404)

The requested resource does not exist.

<AccordionGroup>
  <Accordion title="not_found">
    **Cause:** The resource ID does not exist or does not belong to your account.

    **Fix:** Verify the resource ID is correct and belongs to your account.

    ```json
    {
      "type": "error",
      "error": {
        "type": "not_found_error",
        "code": "not_found",
        "message": "Webhook endpoint not found.",
        "display_message": "The requested resource was not found.",
        "param": "id",
        "doc_url": "https://developers.trynawa.com/errors#not_found",
        "suggested_action": "Check the resource ID and try again."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>
</AccordionGroup>

### `api_error` (500)

An unexpected error occurred on the NAWA side.

<AccordionGroup>
  <Accordion title="internal_error">
    **Cause:** An unexpected internal failure.

    **Fix:** Retry the request. If the issue persists, contact support with the `request_id`.

    ```json
    {
      "type": "error",
      "error": {
        "type": "api_error",
        "code": "internal_error",
        "message": "An unexpected error occurred. Please retry.",
        "display_message": "Something went wrong on our end.",
        "param": null,
        "doc_url": "https://developers.trynawa.com/errors#internal_error",
        "suggested_action": "Retry the request. Contact support if the issue persists."
      },
      "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
    }
    ```
  </Accordion>
</AccordionGroup>

## Handling errors in code

<CodeGroup>

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/classify', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ text: 'متى الجزء الثاني؟' })
})

const data = await response.json()

if (data.type === 'error') {
  switch (data.error.type) {
    case 'authentication_error':
      console.error('Check your API key:', data.error.message)
      break
    case 'rate_limit_error':
      const retryAfter = response.headers.get('Retry-After')
      console.log(`Rate limited, retrying after ${retryAfter}s`)
      break
    case 'permission_error':
      console.error('Buy credits:', data.error.suggested_action)
      break
    default:
      console.error('API error:', data.error.message)
  }
}
```

```python Python
import requests

response = requests.post(
    "https://api.trynawa.com/v1/classify",
    headers={
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    },
    json={"text": "متى الجزء الثاني؟"}
)

data = response.json()

if data.get("type") == "error":
    error = data["error"]
    if error["type"] == "authentication_error":
        print(f"Check your API key: {error['message']}")
    elif error["type"] == "rate_limit_error":
        retry_after = response.headers.get("Retry-After")
        print(f"Rate limited, retry after {retry_after}s")
    elif error["type"] == "permission_error":
        print(f"Buy credits: {error['suggested_action']}")
    else:
        print(f"API error: {error['message']}")
```
