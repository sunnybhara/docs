Classify text against a user-defined rubric. Define your own categories with descriptions for domain-specific classification. Supports multi-label classification and configurable confidence thresholds.

<Note>
  Cost: **3 credits** ($0.003) per request.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify. Must be non-empty. |
| `rubric` | object | Yes | The rubric definition (see below). |
| `rubric.categories` | object[] | Yes | Array of 1-20 category objects. Each must have a `name` (string). May include a `description` (string). |
| `rubric.multi_label` | boolean | No | If `true`, returns all matching categories above the threshold. Default: `false`. |
| `rubric.confidence_threshold` | number | No | Minimum confidence to include a category (0-1). Default: `0.5`. |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider: `claude`, `gemini`, or `allam`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/rubric-classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "وين الترجمة العربية؟ مافهمت شي",
    "rubric": {
      "categories": [
        { "name": "translation_request", "description": "User is asking for subtitles or translation" },
        { "name": "technical_issue", "description": "Audio, video, or playback problems" },
        { "name": "content_feedback", "description": "Comments about the content quality" },
        { "name": "off_topic", "description": "Not related to the video" }
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

```typescript TypeScript

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

      { name: 'content_feedback', description: 'Comments about the content quality' },
      { name: 'off_topic', description: 'Not related to the video' }
    ],
    multiLabel: false,
    confidenceThreshold: 0.5
  }
})
```

```python Python

            {"name": "technical_issue", "description": "Audio, video, or playback problems"},

            {"name": "off_topic", "description": "Not related to the video"},
        ],

```

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_a1b2c3d4e5f6",
    "object": "rubric_classification",
    "categories": [
      { "name": "translation_request", "confidence": 0.94 }
    ],
    "multi_label": false,
    "language": "ar",
    "provider": "claude",
    "model": "claude-haiku-4-5-20251001",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_nw_rcl123abc456"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Rubric classification ID in `rcl_nw_xxx` format |
| `object` | string | Always `"rubric_classification"` |
| `categories` | object[] | Matched categories with `name` and `confidence` score |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language code |
| `provider` | string | AI provider used |
| `model` | string | Model version used |
| `fallback_used` | boolean | Whether the fallback provider was used |
| `cost_usd` | number | Cost in USD |
| `credits_used` | number | Credits deducted |

### Response headers

| Header | Description |
|--------|-------------|
| `X-NAWA-Provider` | AI provider used |
| `X-NAWA-Latency` | Request latency in milliseconds |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text`, missing `rubric`, empty `rubric.categories`, more than 20 categories, category missing `name` |
| 401 | `authentication_error` | Invalid or missing API key |
| 405 | `invalid_request_error` | Wrong HTTP method (only POST allowed) |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
