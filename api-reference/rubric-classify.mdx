Classify text against your own categories. Define a rubric with category names, descriptions, and optional multi-label support for domain-specific classification.

<Note>
  Cost: **$0.003** per request (3 credits). No semantic caching on this endpoint.
</Note>

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The text to classify. Must be non-empty. |
| `rubric` | object | Yes | Rubric definition (see below). |
| `rubric.categories` | array | Yes | Array of category objects. Min 1, max 20. Each object has a required `name` (string) and optional `description` (string). |
| `rubric.multi_label` | boolean | No | Allow multiple category matches. Default: `false`. |
| `rubric.confidence_threshold` | number | No | Minimum confidence to include a category (0-1). Default: `0.5`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST "https://api.trynawa.com/v1/rubric-classify" \
  -H "Authorization: Bearer nawa_live_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "أحتاج مساعدة في طلبي رقم ١٢٣٤",
    "rubric": {
      "categories": [
        {"name": "support_request", "description": "User needs help with an order"},
        {"name": "billing_issue", "description": "Payment or invoice problems"},
        {"name": "general_feedback", "description": "General comments about the product"},
        {"name": "off_topic", "description": "Not related to the product"}
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_a1b2c3d4e5f6",
    "object": "rubric_classification",
    "categories": [
      {"name": "support_request", "confidence": 0.94},
      {"name": "billing_issue", "confidence": 0.12},
      {"name": "general_feedback", "confidence": 0.03},
      {"name": "off_topic", "confidence": 0.01}
    ],
    "multi_label": false,
    "language": "ar",
    "provider": "allam",
    "model": "sdaia/allam-1-13b-instruct",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Rubric classification ID in `rcl_nw_xxx` format |
| `object` | string | Always `rubric_classification` |
| `categories` | array | Array of `{ name, confidence }` objects sorted by confidence |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language: `ar`, `en`, or `mixed` |
| `provider` | string | AI provider used |
| `model` | string | Specific model version |
| `fallback_used` | boolean | Whether a fallback provider was needed |
| `cost_usd` | number | Cost in USD |
| `credits_used` | integer | Credits consumed |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or `rubric`, invalid category format, too many categories |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `permission_error` | Insufficient credits |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
