Classify a comment against a custom rubric. Define your own categories with descriptions for domain-specific classification. Supports single-label and multi-label classification with configurable confidence thresholds.

<Note>
  Cost: **$0.003** per request (3 credits). **500 free requests/month** on this endpoint -- no credit card required.
</Note>

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider for A/B testing: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify. Must be non-empty. |
| `rubric` | object | Yes | The rubric definition (see below). |
| `rubric.categories` | array | Yes | Array of category objects. Each must have a `name` (string). Optional `description` (string) per category. Max 20 categories. |
| `rubric.multi_label` | boolean | No | When `true`, returns all matching categories above the confidence threshold. Default: `false` (single best match). |
| `rubric.confidence_threshold` | number | No | Minimum confidence score (0-1) a category must reach. Default: `0.5`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/rubric/classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "وين الترجمة العربية؟ مافهمت شي",
    "rubric": {
      "categories": [
        {"name": "translation_request", "description": "User is asking for subtitles or translation"},
        {"name": "technical_issue", "description": "Audio, video, or playback problems"},
        {"name": "content_feedback", "description": "Comments about the content quality"},
        {"name": "off_topic", "description": "Not related to the video"}
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_a1b2c3d4e5f6",
    "object": "rubric_classification",
    "categories": [
      {"name": "translation_request", "confidence": 0.94}
    ],
    "multi_label": false,
    "language": "ar",
    "provider": "allam",
    "model": "nagl-v1",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_rub123abc456"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique classification ID (`rcl_` prefix) |
| `object` | string | Always `rubric_classification` |
| `categories` | array | Matched categories, each with `name` and `confidence` (0-1). When `multi_label` is `false`, contains a single entry. |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language code |
| `provider` | string | AI provider used |
| `model` | string | Model version used |
| `fallback_used` | boolean | Whether a fallback provider was used |
| `cost_usd` | number | Cost in USD |
| `credits_used` | integer | Credits deducted |

### Multi-label example

When `multi_label` is `true`, NAWA returns all categories above the confidence threshold:

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_multi001",
    "object": "rubric_classification",
    "categories": [
      {"name": "translation_request", "confidence": 0.88},
      {"name": "content_feedback", "confidence": 0.62}
    ],
    "multi_label": true,
    "language": "ar",
    "provider": "allam",
    "model": "nagl-v1",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_rub_multi001"
}
```

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or `rubric.categories`. Categories exceed 20. Category missing `name`. |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
