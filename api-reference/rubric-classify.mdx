Classify text against a user-defined rubric. Define your own categories and scoring criteria for domain-specific classification. Supports multi-label classification and configurable confidence thresholds.

<Note>
  Cost: **$0.003** per request (3 credits). No semantic caching on this endpoint.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider for A/B testing: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The text to classify. Must be non-empty. |
| `rubric` | object | Yes | Rubric definition (see below). |
| `rubric.categories` | array | Yes | Array of category objects, each with `name` (required) and `description` (optional). 1-20 categories. |
| `rubric.multi_label` | boolean | No | Enable multi-label classification. Default: `false`. |
| `rubric.confidence_threshold` | number | No | Minimum confidence to include a category. 0-1, default: `0.5`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/rubric-classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "أحتاج مساعدة في طلبي رقم ١٢٣٤",
    "rubric": {
      "categories": [
        {"name": "order_support", "description": "Customer needs help with an existing order"},
        {"name": "product_question", "description": "Questions about product features or availability"},
        {"name": "billing", "description": "Payment, refund, or invoice issues"},
        {"name": "general_feedback", "description": "General comments or suggestions"}
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.rubric.classify({
  text: 'أحتاج مساعدة في طلبي رقم ١٢٣٤',
  rubric: {
    categories: [
      { name: 'order_support', description: 'Customer needs help with an existing order' },
      { name: 'product_question', description: 'Questions about product features or availability' },
      { name: 'billing', description: 'Payment, refund, or invoice issues' },
      { name: 'general_feedback', description: 'General comments or suggestions' }
    ],
    multiLabel: false,
    confidenceThreshold: 0.5
  }
})

console.log(result.data.categories) // [{name: "order_support", confidence: 0.94}]
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.rubric.classify(
    text="أحتاج مساعدة في طلبي رقم ١٢٣٤",
    rubric={
        "categories": [
            {"name": "order_support", "description": "Customer needs help with an existing order"},
            {"name": "product_question", "description": "Questions about product features or availability"},
            {"name": "billing", "description": "Payment, refund, or invoice issues"},
            {"name": "general_feedback", "description": "General comments or suggestions"}
        ],
        "multi_label": False,
        "confidence_threshold": 0.5
    }
)

print(result.data.categories)  # [{"name": "order_support", "confidence": 0.94}]
```

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_a1b2c3d4e5f6",
    "object": "rubric_classification",
    "categories": [
      {"name": "order_support", "confidence": 0.94}
    ],
    "multi_label": false,
    "language": "ar",
    "provider": "allam",
    "model": "sdaia/allam-1-13b-instruct",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_nw_rcl1a2b3c4d5e"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Rubric classification ID in `rcl_nw_xxx` format |
| `object` | string | Always `"rubric_classification"` |
| `categories` | array | Matched categories with `name` and `confidence` (0-1). Categories below the threshold are filtered out. |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language: `ar`, `en`, `mixed` |
| `provider` | string | AI provider used: `claude`, `gemini`, `allam` |
| `model` | string | Specific model version used |
| `fallback_used` | boolean | Whether a fallback provider was used |
| `cost_usd` | number | Cost in USD (always 0.003) |
| `credits_used` | integer | Credits deducted (always 3) |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or `rubric`. Empty categories. Category without `name`. More than 20 categories. |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
