Classify text against a user-defined rubric. Define your own categories with optional descriptions for domain-specific classification. Supports multi-label classification and configurable confidence thresholds.

<Note>
  Cost: **$0.003** per request (3 credits). Supports `?provider=` query parameter for A/B testing.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify. Max 5,000 characters. |
| `rubric` | object | Yes | The rubric definition. |
| `rubric.categories` | array | Yes | Array of category objects, each with `name` (string, required) and `description` (string, optional). 1-20 categories. |
| `rubric.multi_label` | boolean | No | Allow multiple categories to be returned. Default: `false`. |
| `rubric.confidence_threshold` | number | No | Minimum confidence to include a category (0-1). Default: `0.5`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/rubric-classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "وين الترجمة العربية؟ مافهمت شي",
    "rubric": {
      "categories": [
        { "name": "translation_request", "description": "User is asking for subtitles or translation" },
        { "name": "technical_issue", "description": "Audio, video, or playback problems" },
        { "name": "content_feedback", "description": "Comments about the content quality" },
        { "name": "off_topic", "description": "Not related to the video" }
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

```python Python

    },
)
```

```json
  "credits_used": 3,
  "cached": false,

```

### Multi-label response

When `multi_label` is `true`, multiple categories can be returned:

```json
{
  "id": "rcl_nw_b2c3d4e5f6g7",
  "object": "rubric_classification",
  "categories": [
    { "name": "translation_request", "confidence": 0.91 },
    { "name": "content_feedback", "confidence": 0.67 }
  ],
  "multi_label": true,
  "language": "ar",
  "provider": "claude",
  "model": "claude-haiku-4-5-20251001",
  "fallback_used": false,
  "cost_usd": 0.003,
  "credits_used": 3,
  "cached": false,
  "request_id": "req_nw_rub456def789"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Rubric classification ID (`rcl_nw_xxx` format) |
| `object` | string | Always `"rubric_classification"` |
| `categories` | array | Matched categories, each with `name` and `confidence` (0-1) |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language: `ar`, `en`, `mixed` |
| `provider` | string | AI provider used: `allam`, `claude`, or `gemini` |
| `model` | string | Specific model version |
| `fallback_used` | boolean | Whether a fallback provider was used |
| `cost_usd` | number | Cost in USD |
| `credits_used` | integer | Credits deducted |
| `cached` | boolean | Whether served from semantic cache |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or `rubric`. Categories empty or exceeds 20. |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
