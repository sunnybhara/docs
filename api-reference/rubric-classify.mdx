Classify text against a custom rubric. Define your own categories with descriptions for domain-specific classification. Supports multi-label classification and configurable confidence thresholds.

<Note>
  Cost: **$0.003** per request (3 credits).
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The text to classify. |
| `rubric` | object | Yes | The rubric definition. |
| `rubric.categories` | object[] | Yes | Array of category objects. Each must have a `name` (string) and an optional `description` (string). Max 20 categories. |
| `rubric.multi_label` | boolean | No | Return all matching categories above the threshold. Default: `false` (returns only the best match). |
| `rubric.confidence_threshold` | number | No | Minimum confidence score (0-1) for a category to match. Default: `0.5`. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/rubric-classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "وين الترجمة العربية؟ مافهمت شي",
    "rubric": {
      "categories": [
        {"name": "translation_request", "description": "User is asking for subtitles or translation"},
        {"name": "technical_issue", "description": "Audio, video, or playback problems"},
        {"name": "content_feedback", "description": "Comments about the content quality"},
        {"name": "off_topic", "description": "Not related to the video"}
      ],
      "multi_label": false,
      "confidence_threshold": 0.5
    }
  }'
```

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "rcl_nw_a1b2c3d4e5f6",
    "object": "rubric_classification",
    "categories": [
      {"name": "translation_request", "confidence": 0.94}
    ],
    "multi_label": false,
    "language": "ar",
    "provider": "allam",
    "model": "sdaia/allam-1-13b-instruct",
    "fallback_used": false,
    "cost_usd": 0.003,
    "credits_used": 3
  },
  "errors": [],
  "request_id": "req_nw_rub123abc456def789"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Classification ID (`rcl_nw_xxx` format) |
| `object` | string | Always `"rubric_classification"` |
| `categories` | object[] | Matching categories, each with `name` (string) and `confidence` (number, 0-1) |
| `multi_label` | boolean | Whether multi-label mode was used |
| `language` | string | Detected language code |
| `provider` | string | AI provider used |
| `model` | string | Model identifier |
| `fallback_used` | boolean | Whether a fallback provider was used |
| `cost_usd` | number | Cost in USD |
| `credits_used` | integer | Credits deducted |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or `rubric`, invalid category format, exceeds 20 categories |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
