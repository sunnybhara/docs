Register webhook endpoints to receive real-time event notifications. This endpoint is **free**.

## Create a webhook

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/webhooks \
  -H "Authorization: Bearer nawa_live_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/webhooks/nawa",
    "events": ["classification.completed", "credits.low"]
  }'
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.webhooks.create({
  url: 'https://example.com/webhooks/nawa',
  events: ['classification.completed', 'credits.low']
})

// Save the signing_secret -- it is only shown once
console.log(data.signing_secret)
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.webhooks.create(
    url="https://example.com/webhooks/nawa",
    events=["classification.completed", "credits.low"]
)

# Save the signing_secret -- it is only shown once
print(result.data.signing_secret)
```

</CodeGroup>

### Requirements

- URL must use HTTPS
- Maximum 10 webhook endpoints per account
- At least one event type required

### Available events

| Event | Description |
|-------|-------------|
| `classification.completed` | A classification request succeeded |
| `classification.failed` | A classification request failed |
| `credits.low` | Credit balance dropped below $5 |
| `credits.exhausted` | Credit balance reached $0 |

### Response (201)

```json
{
  "success": true,
  "result": {
    "id": "wh_abc123",
    "url": "https://example.com/webhooks/nawa",
    "events": ["classification.completed", "credits.low"],
    "active": true,
    "signing_secret": "nawa_wh_a1b2c3d4e5f6...",
    "created_at": "2025-01-15T10:00:00Z"
  },
  "errors": [],
  "request_id": "req_wh_abc123"
}
```

<Warning>
  The `signing_secret` is shown **once** at creation. Store it securely to verify webhook signatures.
</Warning>

## List webhooks

```bash
curl https://api.trynawa.com/v1/webhooks \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

### Response (200)

```json
{
  "success": true,
  "result": [
    {
      "id": "wh_abc123",
      "url": "https://example.com/webhooks/nawa",
      "events": ["classification.completed", "credits.low"],
      "active": true,
      "failure_count": 0,
      "last_delivery_at": "2025-01-15T12:30:00Z",
      "created_at": "2025-01-15T10:00:00Z"
    }
  ],
  "errors": [],
  "request_id": "req_wh_def456"
}
```

## Deactivate a webhook

```bash
curl -X DELETE "https://api.trynawa.com/v1/webhooks?id=wh_abc123" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

Deactivated webhooks stop receiving events immediately. This is a soft delete -- the endpoint record is preserved with `active: false`.
