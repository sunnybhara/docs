Classify a comment and generate a contextual reply in a single call. The reply matches the commenter's dialect and cultural context. Supports tone control and max length configuration.

<Note>
  Cost: **$0.004** per request.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify and reply to. Minimum 1 character. |
| `tone` | string | No | Reply tone: `friendly`, `professional`, `casual`, `formal`. Default: `professional`. |
| `max_length` | integer | No | Maximum reply length in characters (1 to 2000). Default: 500. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/comments/reply \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "هذا المنتج سيء جداً ولا أنصح به",
    "tone": "professional",
    "max_length": 500
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.trynawa.com/v1/comments/reply",
    headers={
        "Authorization": "Bearer nawa_test_sk_xxx",
        "Content-Type": "application/json",
    },
    json={
        "text": "هذا المنتج سيء جداً ولا أنصح به",
        "tone": "professional",
        "max_length": 500,
    },
)

result = response.json()
print(result["result"]["reply"]["text"])
```

```typescript TypeScript
const response = await fetch("https://api.trynawa.com/v1/comments/reply", {
  method: "POST",
  headers: {
    Authorization: "Bearer nawa_test_sk_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    text: "هذا المنتج سيء جداً ولا أنصح به",
    tone: "professional",
    max_length: 500,
  }),
});

const { result } = await response.json();
console.log(result.reply.text);
```

</CodeGroup>

```json
  "success": true,

    "object": "comment_reply",
    "classification": {
      "intent": ["complaint"],
      "sentiment": "negative",
      "priority": "high",
      "requires_response": true

    "reply": {
      "text": "نعتذر عن تجربتك السيئة. نود معرفة المزيد عن المشكلة لتحسين خدمتنا. تواصل معنا مباشرة وسنعمل على حلها.",
      "direction": "rtl",

    "provider": "allam",
    "model": "sdaia/allam-1-13b-instruct",
    "cost": 0.004
  },
  "errors": [],
  "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"

```

### Response headers

| Header | Description |
|--------|-------------|
| `X-Request-Id` | Unique request identifier (`req_nw_xxx` format) |
| `X-RateLimit-Limit` | Rate limit ceiling for current window |
| `X-RateLimit-Remaining` | Requests remaining in current window |
| `X-RateLimit-Reset` | Window reset time (RFC 3339) |
| `X-NAWA-Provider` | AI provider used for this request |

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Reply ID (`rpl_nw_xxx` format) |
| `object` | string | Always `comment_reply` |
| `classification.intent` | string[] | Detected intents of the original comment |
| `classification.sentiment` | string | `positive`, `negative`, `neutral`, `mixed` |
| `classification.priority` | string | `high`, `medium`, `low` |
| `classification.requires_response` | boolean | Whether the comment warrants a reply |
| `reply.text` | string | The generated reply text |
| `reply.direction` | string | Text direction: `ltr` or `rtl` |
| `reply.tone` | string | The tone used for the reply |
| `provider` | string | AI provider used |
| `model` | string | Model identifier |
| `cost` | number | Cost in USD |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text`, invalid `tone` value |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `permission_error` | Insufficient credits |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
