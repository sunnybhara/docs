Classify and generate an AI-powered reply in a single request. The response includes the classification (intent, sentiment, priority) alongside a dialect-matched reply. Replies are contextually appropriate for the detected platform.

<Note>
  Cost: **$0.008** per request (8 credits). Semantic cache hits are free (`X-NAWA-Cache: HIT`).
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Override the AI provider: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify and reply to. Max 5,000 characters. |
| `tone` | string | No | Reply tone: `friendly`, `professional`, `casual`, `formal`. Default: `friendly`. |
| `max_length` | integer | No | Maximum reply length in characters (1-2000). Default: 500. |
| `context` | object | No | Additional context to improve reply relevance. |
| `context.platform` | string | No | Source platform: `youtube`, `instagram`, `twitter`, `facebook`. |
| `context.brand_voice` | string | No | Description of your brand voice for tone matching. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/comments/reply \
  -H "Authorization: Bearer $NAWA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "متى الجزء الثاني؟",
    "tone": "friendly",
    "context": {
      "platform": "youtube",
      "brand_voice": "Tech review channel focused on smartphones"
    }
  }'
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.comments.reply({
  text: 'متى الجزء الثاني؟',
  tone: 'friendly',
  context: {
    platform: 'youtube',
    brandVoice: 'Tech review channel focused on smartphones'
  }
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.comments.reply(
    text="متى الجزء الثاني؟",
    tone="friendly",
    context={
        "platform": "youtube",
        "brand_voice": "Tech review channel focused on smartphones"
    }
)
```

</CodeGroup>

## Response

### Success response (200)

```json
{
  "result": {
    "id": "rep_nw_pon1yom9bkfe",
    "object": "comment_reply",
    "classification": {
      "intent": ["question"],
      "sentiment": "neutral",
      "priority": "medium",
      "requires_response": true
    },
    "reply": {
      "text": "إن شاء الله الجزء الثاني قريب! تابعنا عشان ما يفوتك",
      "direction": "rtl",
      "tone": "friendly"
    },
    "provider": "allam",
    "model": "allam-v2",
    "cost_usd": 0.008,
    "credits_used": 8
  },
  "success": true,
  "errors": [],
  "request_id": "req_nw_rep789xyz012"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique reply ID (format: `rep_nw_xxx`) |
| `object` | string | Always `"comment_reply"` |
| `classification.intent` | string[] | Detected intents of the original comment |
| `classification.sentiment` | string | Detected sentiment of the original comment |
| `classification.priority` | string | Priority level: `low`, `medium`, `high` |
| `classification.requires_response` | boolean | Whether the comment needs a reply |
| `reply.text` | string | The generated reply text |
| `reply.direction` | string | Text direction: `rtl` or `ltr` |
| `reply.tone` | string | The tone used for the reply |
| `provider` | string | AI provider used |
| `model` | string | Model version used |
| `cost_usd` | number | Cost in USD |
| `credits_used` | number | Credits deducted |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text`. Invalid tone. Text too long. `max_length` out of range. |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
