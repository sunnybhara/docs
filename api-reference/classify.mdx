Classify an Arabic or English comment with a single request. Returns intent, sentiment, dialect, priority, and a suggested reply direction powered by HUMAIN's ALLaM model.

## Request

<Note>
  Cost: **$0.006** per request (6 credits). Semantic cache hits are free (`X-NAWA-Cache: HIT`).
</Note>

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |
| `Content-Type` | Yes | `application/json` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `provider` | string | No | Force a specific AI provider for A/B testing: `claude`, `gemini`, `allam`. |

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | The comment text to classify. Must be non-empty. |
| `context` | object | No | Optional context object for classification. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/classify \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "ماشاء الله الفيديو حلو مرة"
  }'
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.classify({
  text: 'ماشاء الله الفيديو حلو مرة'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.classify(
    text="ماشاء الله الفيديو حلو مرة"
)
```

</CodeGroup>

### Response headers

| Header | Description |
|--------|-------------|
| `X-Request-Id` | Unique request identifier for debugging |
| `X-RateLimit-Limit` | Rate limit ceiling for current window |
| `X-RateLimit-Remaining` | Requests remaining in current window |
| `X-RateLimit-Reset` | Window reset time (RFC 3339) |
| `X-NAWA-Cache` | `HIT` if served from semantic cache (no cost), `MISS` otherwise |
| `X-NAWA-Provider` | AI provider used for this request |
| `X-NAWA-Latency` | Processing latency in milliseconds |
| `X-NAWA-Calibration-Version` | Calibration version applied to results |

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Classification ID in `cls_nw_xxx` format |
| `object` | string | Always `"classification"` |
| `intent` | string[] | Detected intents. Values: `question`, `feedback`, `complaint`, `praise`, `spam`, `collaboration`, `other` |
| `sentiment` | string | `positive`, `negative`, `neutral`, `mixed` |
| `language` | string | Detected language: `ar`, `en`, or `mixed` |
| `dialect` | string or null | Arabic dialect when language is Arabic. Null for non-Arabic text. |
| `dialect_confidence` | number or null | Confidence score (0-1) for dialect detection. Null for non-Arabic text. |
| `requires_response` | boolean | Whether the comment needs a reply |
| `priority` | string | `high`, `medium`, `low` |
| `suggested_reply` | object | Contains `text` (suggested reply direction) and `direction` (`ltr` or `rtl`) |
| `provider` | string | AI provider used: `allam`, `claude`, or `gemini` |
| `model` | string | Model identifier |
| `fallback_used` | boolean | Whether a fallback provider was needed |
| `tokens_used` | integer or null | Token count (currently null) |
| `cost_usd` | number | Cost in USD for this request |
| `credits_used` | number | Credits deducted |

<Info>
  The `intent` field returns an **array of strings**, supporting multi-intent classification. For example, a comment can be both a `question` and a `complaint`.
</Info>

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `text` or empty text |
| 401 | `authentication_error` | Invalid or missing API key |
| 402 | `insufficient_credits` | No credits remaining |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal or provider error |
