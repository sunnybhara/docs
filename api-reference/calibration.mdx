Customize classification confidence thresholds for sentiment, intent, and dialect detection. When calibration is active, results from both cached and fresh classifications are adjusted to match your thresholds.

<Note>
  Calibration is available on **Enterprise** and **Enterprise Plus** tiers only. Free and live tier users receive a `403 tier_insufficient` error.
</Note>

## Get calibration

<br />

`GET /v1/calibration`

Retrieve your current calibration thresholds, defaults, and recent change history.

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/calibration" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
const response = await fetch("https://api.trynawa.com/v1/calibration", {
  headers: { Authorization: "Bearer nawa_live_sk_xxx" },
})

const data = await response.json()
```

```python Python
import requests

response = requests.get(
    "https://api.trynawa.com/v1/calibration",
    headers={"Authorization": "Bearer nawa_live_sk_xxx"},
)

data = response.json()
```

</CodeGroup>

### Success response (200)

```json
{
  "success": true,
  "result": {
    "calibration": {
      "sentiment_threshold": 0.7,
      "intent_threshold": 0.6,
      "dialect_threshold": 0.5
    },
    "defaults": {
      "sentiment_threshold": 0.7,
      "intent_threshold": 0.6,
      "dialect_threshold": 0.5
    },
    "history": [
      {
        "updated_at": "2025-01-10T14:30:00Z",
        "changes": {"sentiment_threshold": 0.8}
      }
    ]
  },
  "errors": [],
  "request_id": "req_nw_cal_abc123"
}
```

## Update calibration

<br />

`POST /v1/calibration`

Partially update calibration thresholds. Only include the fields you want to change.

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `sentiment_threshold` | number | No | Confidence threshold for sentiment (0--1). Default: 0.7. |
| `intent_threshold` | number | No | Confidence threshold for intent (0--1). Default: 0.6. |
| `dialect_threshold` | number | No | Confidence threshold for dialect (0--1). Default: 0.5. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST "https://api.trynawa.com/v1/calibration" \
  -H "Authorization: Bearer nawa_live_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{"sentiment_threshold": 0.8}'
```

```typescript TypeScript
const response = await fetch("https://api.trynawa.com/v1/calibration", {
  method: "POST",
  headers: {
    Authorization: "Bearer nawa_live_sk_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ sentiment_threshold: 0.8 }),
})

const data = await response.json()
```

```python Python
import requests

response = requests.post(
    "https://api.trynawa.com/v1/calibration",
    headers={
        "Authorization": "Bearer nawa_live_sk_xxx",
        "Content-Type": "application/json",
    },
    json={"sentiment_threshold": 0.8},
)

data = response.json()
```

</CodeGroup>

### Success response (200)

```json
{
  "success": true,
  "result": {
    "calibration": {
      "sentiment_threshold": 0.8,
      "intent_threshold": 0.6,
      "dialect_threshold": 0.5
    }
  },
  "errors": [],
  "request_id": "req_nw_cal_def456"
}
```

## Reset calibration

<br />

`DELETE /v1/calibration`

Reset all calibration thresholds to defaults.

### Example request

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.trynawa.com/v1/calibration" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
const response = await fetch("https://api.trynawa.com/v1/calibration", {
  method: "DELETE",
  headers: { Authorization: "Bearer nawa_live_sk_xxx" },
})

const data = await response.json()
```

```python Python
import requests

response = requests.delete(
    "https://api.trynawa.com/v1/calibration",
    headers={"Authorization": "Bearer nawa_live_sk_xxx"},
)

data = response.json()
```

</CodeGroup>

### Success response (200)

```json
{
  "success": true,
  "result": {
    "reset": true
  },
  "errors": [],
  "request_id": "req_nw_cal_ghi789"
}
```

## Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid threshold values (must be 0--1) |
| 401 | `authentication_error` | Invalid or missing API key |
| 403 | `permission_error` / `tier_insufficient` | Account is not on Enterprise or Enterprise Plus tier |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
