The calibration endpoint lets you tune classification confidence thresholds for your account. This is an Enterprise-tier feature that gives you fine-grained control over how NAWA interprets classification results.

<Warning>
  Calibration is available on the Enterprise and Enterprise+ tiers only. Other tiers receive a `403 tier_insufficient` error.
</Warning>

## Get calibration

`GET https://api.trynawa.com/v1/calibration`

Retrieve your current calibration thresholds, defaults, and change history.

### Example request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/calibration \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/calibration', {
  headers: { Authorization: `Bearer ${process.env.NAWA_API_KEY}` }
})
const { result } = await response.json()
console.log(result.calibration)
```

```python Python
import requests

response = requests.get(
    "https://api.trynawa.com/v1/calibration",
    headers={"Authorization": "Bearer nawa_live_sk_xxx"}
)
result = response.json()["result"]
print(result["calibration"])
```

</CodeGroup>

### Success response (200)

```json
{
  "result": {
    "calibration": {
      "sentiment_threshold": 0.7,
      "intent_threshold": 0.6,
      "dialect_threshold": 0.5
    },
    "defaults": {
      "sentiment_threshold": 0.7,
      "intent_threshold": 0.6,
      "dialect_threshold": 0.5
    },
    "history": []
  },
  "success": true,
  "errors": [],
  "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
}
```

## Update calibration

`POST https://api.trynawa.com/v1/calibration`

Partially update your calibration thresholds. Only include the fields you want to change.

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `sentiment_threshold` | number | No | Confidence threshold for sentiment (0-1) |
| `intent_threshold` | number | No | Confidence threshold for intent (0-1) |
| `dialect_threshold` | number | No | Confidence threshold for dialect (0-1) |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/calibration \
  -H "Authorization: Bearer nawa_live_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "sentiment_threshold": 0.8,
    "intent_threshold": 0.7
  }'
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/calibration', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${process.env.NAWA_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    sentiment_threshold: 0.8,
    intent_threshold: 0.7
  })
})
```

```python Python
import requests

response = requests.post(
    "https://api.trynawa.com/v1/calibration",
    headers={
        "Authorization": "Bearer nawa_live_sk_xxx",
        "Content-Type": "application/json"
    },
    json={
        "sentiment_threshold": 0.8,
        "intent_threshold": 0.7
    }
)
```

</CodeGroup>

### Success response (200)

```json
{
  "result": {
    "calibration": {
      "sentiment_threshold": 0.8,
      "intent_threshold": 0.7,
      "dialect_threshold": 0.5
    }
  },
  "success": true,
  "errors": [],
  "request_id": "req_nw_b2c3d4e5f6a7b8c9d0e1f2a3"
}
```

## Reset calibration

`DELETE https://api.trynawa.com/v1/calibration`

Reset all calibration thresholds to defaults.

### Example request

<CodeGroup>

```bash cURL
curl -X DELETE https://api.trynawa.com/v1/calibration \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/calibration', {
  method: 'DELETE',
  headers: { Authorization: `Bearer ${process.env.NAWA_API_KEY}` }
})
```

```python Python
import requests

response = requests.delete(
    "https://api.trynawa.com/v1/calibration",
    headers={"Authorization": "Bearer nawa_live_sk_xxx"}
)
```

</CodeGroup>

### Success response (200)

```json
{
  "result": {
    "reset": true
  },
  "success": true,
  "errors": [],
  "request_id": "req_nw_c3d4e5f6a7b8c9d0e1f2a3b4"
}
```

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid threshold values |
| 401 | `authentication_error` | Invalid or missing API key |
| 403 | `permission_error` | Account tier does not include calibration |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |

### How calibration affects classification

When you set a calibration threshold, the `X-NAWA-Calibration-Version` response header on `/v1/classify` requests reflects your current calibration version. Classification results are filtered through your thresholds before being returned.

For example, if you set `dialect_threshold: 0.8`, dialect results with confidence below 0.8 may be adjusted or omitted from the response.

<Note>
  Calibration settings are applied per-user. Cached classification results are re-calibrated at response time, so changing your thresholds takes effect immediately without invalidating the cache.
</Note>
