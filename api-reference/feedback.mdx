Submit feedback on a classification result to continuously improve NAWA's accuracy. Feedback is stored and processed through the RLHF pipeline.

<Note>
  This endpoint is **free** -- 0 credits, no cost.
</Note>

<Tip>
  Every feedback submission helps improve accuracy for all NAWA users. We especially value feedback on dialect detection edge cases.
</Tip>

## Request

### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `classification_id` | string | Yes | The `id` from the original classification response (e.g., `cls_nw_abc123`). |
| `rating` | string | Yes | How accurate the classification was: `correct`, `incorrect`, `partial`. |
| `corrected_intent` | string[] | No | Array of correct intent values if the classification was wrong. |
| `corrected_sentiment` | string | No | Correct sentiment if wrong: `positive`, `negative`, `neutral`, `mixed`. |
| `comment` | string | No | Free-text explanation of why this correction is needed. |

### Example request

<CodeGroup>

```bash cURL
curl -X POST https://api.trynawa.com/v1/feedback \
  -H "Authorization: Bearer nawa_test_sk_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "classification_id": "cls_nw_abc123def456",
    "rating": "incorrect",
    "corrected_intent": ["suggestion"],
    "corrected_sentiment": "neutral",
    "comment": "This is a content suggestion, not a question"
  }'
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.feedback.submit({
  classificationId: 'cls_nw_abc123def456',
  rating: 'incorrect',
  correctedIntent: ['suggestion'],
  correctedSentiment: 'neutral',
  comment: 'This is a content suggestion, not a question'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.feedback.submit(
    classification_id="cls_nw_abc123def456",
    rating="incorrect",
    corrected_intent=["suggestion"],
    corrected_sentiment="neutral",
    comment="This is a content suggestion, not a question"
)
```

</CodeGroup>

## Response

### Success response (200)

```json
{
  "success": true,
  "result": {
    "id": "fb_nw_xyz789",
    "object": "feedback",
    "classification_id": "cls_nw_abc123def456",
    "rating": "incorrect",
    "acknowledged": true
  },
  "errors": [],
  "request_id": "req_fb_abc123"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique feedback ID (prefixed `fb_`) |
| `object` | string | Always `"feedback"` |
| `classification_id` | string | The classification this feedback refers to |
| `rating` | string | The rating you submitted |
| `acknowledged` | boolean | Always `true` on success |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Missing `classification_id`, invalid `rating`, invalid `corrected_sentiment` |
| 401 | `authentication_error` | Invalid or missing API key |
| 405 | `method_not_allowed` | Using a method other than POST |
| 500 | `api_error` | Internal error |
