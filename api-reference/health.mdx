Check the operational status of the NAWA API. This endpoint is **free** and does **not** require authentication. Use it for uptime monitoring.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.health.check()

if (result.data.status === 'healthy') {
  console.log('All systems operational')
}
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.health.check()

if result.data.status == "healthy":
    print("All systems operational")
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "success": true,
  "result": {
    "status": "healthy",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 12
      },
      "redis": {
        "status": "healthy",
        "latency_ms": 3
      },
      "nagl": {
        "status": "healthy"
      }
    },
    "timestamp": "2025-01-15T12:00:00Z",
    "latency_ms": 15
  },
  "errors": [],
  "request_id": "req_nw_hlt1a2b3c4d5e"
}
```

### Degraded response (503)

When one or more services are degraded or down, the API returns HTTP 503:

```json
{
  "success": true,
  "result": {
    "status": "degraded",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 15
      },
      "redis": {
        "status": "down",
        "latency_ms": 0
      },
      "nagl": {
        "status": "healthy"
      }
    },
    "timestamp": "2025-01-15T12:00:00Z",
    "latency_ms": 18
  },
  "errors": [],
  "request_id": "req_nw_hlt2b3c4d5e6f"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall API status: `healthy` or `degraded` |
| `version` | string | API version (always `v1`) |
| `services` | object | Individual service statuses |
| `services.database` | object | Database status with `status` and `latency_ms` |
| `services.redis` | object | Redis cache status with `status` and `latency_ms` |
| `services.nagl` | object | NAGL language gateway status with `status` |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | integer | Total health check latency in milliseconds |

Service `status` values: `healthy`, `degraded`, `down`.

<Note>
  The health endpoint sets `Cache-Control: no-cache, no-store` to prevent stale status data. Database, Redis, and NAGL are checked in parallel.
</Note>

<Tip>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Tip>
