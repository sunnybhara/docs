Check the operational status of the NAWA API. This endpoint is **free** and does **not** require authentication.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.health.check()
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.health.check()
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "success": true,
  "result": {
    "status": "healthy",
    "version": "v1",
    "services": {
      "database": "operational",
      "redis": "operational",
      "nagl": "operational"
    },
    "timestamp": "2026-03-29T12:00:00Z",
    "latency_ms": 45
  },
  "errors": [],
  "request_id": "req_nw_hlt_abc123def456abcd"
}
```

### Degraded response (503)

```json
{
  "success": true,
  "result": {
    "status": "degraded",
    "version": "v1",
    "services": {
      "database": "operational",
      "redis": "degraded",
      "nagl": "operational"
    },
    "timestamp": "2026-03-29T12:00:00Z",
    "latency_ms": 1200
  },
  "errors": [],
  "request_id": "req_nw_hlt_def456ghi789abcd"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `healthy` or `degraded` |
| `version` | string | API version (`v1`) |
| `services.database` | string | Supabase/Postgres status: `operational`, `degraded`, `down` |
| `services.redis` | string | Redis cache status: `operational`, `degraded`, `down` |
| `services.nagl` | string | NAGL classification engine status: `operational`, `degraded`, `down` |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | integer | Total health check latency in milliseconds |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
