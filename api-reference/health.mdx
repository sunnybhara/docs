Check the operational status of the NAWA API and its backend services. This endpoint is **free** and does **not** require authentication.

Returns HTTP 200 when all services are healthy, or HTTP 503 when one or more services are degraded or down.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const data = await response.json()

if (data.status === 'healthy') {
  console.log('All services operational')
}
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
data = response.json()

if data["status"] == "healthy":
    print("All services operational")
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 12
    },
    "redis": {
      "status": "healthy",
      "latency_ms": 3
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2025-01-15T12:00:00.000Z",
  "latency_ms": 15
}
```

### Degraded response (503)

```json
{
  "status": "degraded",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 12
    },
    "redis": {
      "status": "down",
      "latency_ms": 0
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2025-01-15T12:00:00.000Z",
  "latency_ms": 15
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall status: `healthy` or `degraded` |
| `version` | string | API version (always `v1`) |
| `services.database.status` | string | Database status: `healthy`, `degraded`, or `down` |
| `services.database.latency_ms` | integer | Database response time in ms |
| `services.redis.status` | string | Redis cache status: `healthy`, `degraded`, or `down` |
| `services.redis.latency_ms` | integer | Redis response time in ms |
| `services.nagl.status` | string | NAGL classification engine status: `healthy` or `down` |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | integer | Total health check latency in ms |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
