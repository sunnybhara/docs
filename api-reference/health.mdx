Check the operational status of the NAWA API. This endpoint is **free** and does **not** require authentication.

Returns the status of three services: database (Supabase), Redis cache, and the NAGL classification engine.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const health = await response.json()
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
health = response.json()
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 12
    },
    "redis": {
      "status": "healthy",
      "latency_ms": 3
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2026-01-15T12:00:00Z",
  "latency_ms": 15
}
```

### Degraded response (503)

```json
{
  "status": "degraded",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 15
    },
    "redis": {
      "status": "down",
      "latency_ms": 5000
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2026-01-15T12:00:00Z",
  "latency_ms": 5015
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall status: `healthy` (all services up) or `degraded` (one or more services down) |
| `version` | string | API version. Always `v1`. |
| `services.database` | object | Supabase database status and latency |
| `services.redis` | object | Redis cache status and latency |
| `services.nagl` | object | NAGL classification engine status |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | integer | Total time for all health checks in milliseconds |

Each service object contains a `status` field (`healthy`, `degraded`, or `down`) and an optional `latency_ms` field.

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
