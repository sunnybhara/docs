Check the operational status of the NAWA API. This endpoint is **free** and does **not** require authentication.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.health()
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.health()
```

</CodeGroup>

## Response

### Healthy response (200)

All services are operational.

```json
{
  "success": true,
  "result": {
    "status": "healthy",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 12
      },
      "redis": {
        "status": "healthy",
        "latency_ms": 3
      },
      "nagl": {
        "status": "healthy",
        "latency_ms": 0
      }
    },
    "timestamp": "2026-04-06T12:00:00Z",
    "latency_ms": 15
  },
  "errors": [],
  "request_id": "req_nw_hlt_abc123def456"
}
```

### Degraded response (503)

One or more services are degraded or down.

```json
{
  "success": true,
  "result": {
    "status": "degraded",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 15
      },
      "redis": {
        "status": "down",
        "latency_ms": 0
      },
      "nagl": {
        "status": "healthy",
        "latency_ms": 0
      }
    },
    "timestamp": "2026-04-06T12:00:00Z",
    "latency_ms": 15
  },
  "errors": [],
  "request_id": "req_nw_hlt_def456ghi789"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall status: `healthy` or `degraded` |
| `version` | string | API version (always `v1`) |
| `services` | object | Status of each dependency |
| `services.database` | object | PostgreSQL (Supabase) status and latency |
| `services.redis` | object | Redis cache status and latency |
| `services.nagl` | object | NAGL classification engine status |
| `timestamp` | string | ISO 8601 timestamp |
| `latency_ms` | integer | Total health check latency in milliseconds |

Each service object contains:

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `healthy`, `degraded`, or `down` |
| `latency_ms` | integer | Response time in milliseconds |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
