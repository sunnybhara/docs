---

Check the operational status of the NAWA API and all upstream services. This endpoint is **free** and does **not** require authentication. Returns HTTP 200 when all services are healthy and HTTP 503 when any service is degraded or down.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const health = await response.json()
console.log(health.result.status)
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
health = response.json()
print(health["result"]["status"])
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "success": true,
  "result": {
    "status": "healthy",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 12
      },
      "redis": {
        "status": "healthy",
        "latency_ms": 5
      },
      "nagl": {
        "status": "healthy"
      }
    },
    "timestamp": "2025-01-15T12:00:00Z",
    "latency_ms": 25
  },
  "errors": [],
  "request_id": "req_hlt_abc123"
}
```

### Degraded response (503)

When any service is degraded or down, the endpoint returns HTTP 503:

```json
{
  "success": true,
  "result": {
    "status": "degraded",
    "version": "v1",
    "services": {
      "database": {
        "status": "healthy",
        "latency_ms": 15
      },
      "redis": {
        "status": "down",
        "latency_ms": null
      },
      "nagl": {
        "status": "healthy"
      }
    },
    "timestamp": "2025-01-15T12:00:00Z",
    "latency_ms": 30
  },
  "errors": [],
  "request_id": "req_hlt_def456"
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `healthy` or `degraded` |
| `version` | string | API version |
| `services.database` | object | Database status with `status` and `latency_ms` |
| `services.redis` | object | Redis cache status with `status` and `latency_ms` |
| `services.nagl` | object | NAGL classification engine status |
| `timestamp` | string | ISO 8601 timestamp |
| `latency_ms` | number | Total health check latency in milliseconds |

Service status values: `healthy`, `degraded`, `down`.

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
