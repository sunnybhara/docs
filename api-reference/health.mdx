Check the operational status of the NAWA API. This endpoint is **free** and does **not** require authentication.

<Note>
  The health endpoint returns a raw JSON response (not wrapped in the standard success/errors envelope). Returns HTTP 200 when all services are healthy, or HTTP 503 when any service is degraded or down.
</Note>

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const health = await response.json()

if (health.status === 'healthy') {
  console.log('All systems operational')
} else {
  console.log('Some services degraded:', health.services)
}
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
health = response.json()

if health["status"] == "healthy":
    print("All systems operational")
else:
    print(f"Some services degraded: {health['services']}")
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": { "status": "healthy", "latency_ms": 5 },
    "redis": { "status": "healthy", "latency_ms": 2 },
    "nagl": { "status": "healthy" }
  },
  "timestamp": "2025-01-15T12:00:00.000Z",
  "latency_ms": 12
}
```

### Degraded response (503)

```json
{
  "status": "degraded",
  "version": "v1",
  "services": {
    "database": { "status": "healthy", "latency_ms": 5 },
    "redis": { "status": "down", "latency_ms": 0 },
    "nagl": { "status": "healthy" }
  },
  "timestamp": "2025-01-15T12:00:00.000Z",
  "latency_ms": 8
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall status: `healthy` or `degraded` |
| `version` | string | API version |
| `services.database` | object | Database health with `status` and `latency_ms` |
| `services.redis` | object | Redis cache health with `status` and `latency_ms` |
| `services.nagl` | object | NAGL engine health with `status` |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | number | Total health check latency in milliseconds |

Service statuses: `healthy`, `degraded`, or `down`.

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
