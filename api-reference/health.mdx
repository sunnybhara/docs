Check the operational status of the NAWA API and its dependencies. This endpoint is **free** and does **not** require authentication.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const health = await response.json()
console.log(health.status) // "healthy"
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
health = response.json()
print(health["status"])  # "healthy"
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 4
    },
    "redis": {
      "status": "healthy",
      "latency_ms": 2
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2026-03-17T12:00:00Z",
  "latency_ms": 8
}
```

### Degraded response (503)

Returned when one or more services are not healthy.

```json
{
  "status": "degraded",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 5
    },
    "redis": {
      "status": "down",
      "latency_ms": 1000
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2026-03-17T12:00:00Z",
  "latency_ms": 1005
}
```

### Service status values

| Status | Description |
|--------|-------------|
| `healthy` | Service is operating normally |
| `degraded` | Service is responding but with issues |
| `down` | Service is not responding |

### Services monitored

| Service | Description |
|---------|-------------|
| `database` | Supabase PostgreSQL database connectivity |
| `redis` | Redis cache connectivity |
| `nagl` | NAGL classification engine availability |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
