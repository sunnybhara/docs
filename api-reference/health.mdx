Check the operational status of the NAWA API and its dependencies. This endpoint is **free** and does **not** require authentication.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```typescript TypeScript
const response = await fetch("https://api.trynawa.com/v1/health");
const health = await response.json();
console.log(health.status); // "healthy" or "degraded"
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
health = response.json()
print(health["status"])
```

</CodeGroup>

## Response

### Healthy response (200)

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 15
    },
    "redis": {
      "status": "healthy",
      "latency_ms": 5
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2025-01-15T12:00:00Z",
  "latency_ms": 25
}
```

### Degraded response (503)

When one or more services are unhealthy, the API returns HTTP 503:

```json
{
  "status": "degraded",
  "version": "v1",
  "services": {
    "database": {
      "status": "healthy",
      "latency_ms": 12
    },
    "redis": {
      "status": "down",
      "latency_ms": null
    },
    "nagl": {
      "status": "healthy"
    }
  },
  "timestamp": "2025-01-15T12:05:00Z",
  "latency_ms": 5015
}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall status: `healthy` (all services up) or `degraded` (one or more services down) |
| `version` | string | API version |
| `services.database.status` | string | Database status: `healthy`, `degraded`, or `down` |
| `services.database.latency_ms` | number | Database response time in milliseconds |
| `services.redis.status` | string | Redis cache status: `healthy`, `degraded`, or `down` |
| `services.redis.latency_ms` | number or null | Redis response time in milliseconds |
| `services.nagl.status` | string | NAGL classification engine status: `healthy` or `down` |
| `timestamp` | string | ISO 8601 timestamp of the check |
| `latency_ms` | number | Total health check latency |

### HTTP status codes

| Status | Meaning |
|--------|---------|
| 200 | All services healthy |
| 503 | One or more services degraded or down |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
