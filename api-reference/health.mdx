Check the operational status of the NAWA API and its backend services. This endpoint is **free** and does **not** require authentication or rate limiting.

## Request

<CodeGroup>

```bash cURL
curl https://api.trynawa.com/v1/health
```

```python Python
import requests

response = requests.get("https://api.trynawa.com/v1/health")
print(response.json())
```

```typescript TypeScript
const response = await fetch('https://api.trynawa.com/v1/health')
const health = await response.json()
console.log(health)
```

</CodeGroup>

## Response

### Healthy response (200)

All backend services are operational:

```json
{
  "status": "healthy",
  "version": "v1",
  "services": {
    "database": { "status": "healthy", "latency_ms": 12 },
    "redis": { "status": "healthy", "latency_ms": 5 },
    "nagl": { "status": "healthy" }
  },
  "timestamp": "2025-01-15T12:00:00Z",
  "latency_ms": 15,
  "request_id": "req_nw_hlt_abc123"
}
```

### Degraded response (503)

One or more services are unhealthy. The API returns HTTP 503 when any service is down:

```json
{
  "status": "unhealthy",
  "version": "v1",
  "services": {
    "database": { "status": "healthy", "latency_ms": 15 },
    "redis": { "status": "unhealthy", "latency_ms": null },
    "nagl": { "status": "healthy" }
  },
  "timestamp": "2025-01-15T12:00:00Z",
  "latency_ms": 2015,
  "request_id": "req_nw_hlt_def456"
}
```

### Services monitored

| Service | Description |
|---------|-------------|
| `database` | Supabase PostgreSQL connectivity |
| `redis` | Upstash Redis (semantic cache and rate limiting) |
| `nagl` | NAWA's NAGL classification engine configuration |

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `healthy` or `unhealthy` |
| `version` | string | API version |
| `services` | object | Per-service health with `status` and `latency_ms` |
| `timestamp` | string | ISO 8601 timestamp of the health check |
| `latency_ms` | number | Total health check duration in milliseconds |

<Note>
  For real-time status monitoring, visit [status.trynawa.com](https://status.trynawa.com).
</Note>
