Retrieve aggregated analytics across your API usage. Includes request counts, costs, cache hit rates, latency, and provider distribution.

<Note>
  This endpoint is **free** -- 0 credits, no cost.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601) |
| `to` | string | No | End date (ISO 8601) |
| `group_by` | string | No | Group results by: `endpoint`, `day`, `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2025-01-01T00:00:00Z&to=2025-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from "@nawalabs/sdk";

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY });

const result = await nawa.analytics.get({
  from: "2025-01-01T00:00:00Z",
  to: "2025-01-31T23:59:59Z",
  groupBy: "endpoint",
});

console.log(result.analytics.total_requests);
console.log(result.analytics.cache_hit_rate);
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="nawa_live_sk_xxx")

result = nawa.analytics.get(
    from_date="2025-01-01T00:00:00Z",
    to_date="2025-01-31T23:59:59Z",
    group_by="endpoint",
)

print(result.analytics["total_requests"])
print(result.analytics["cache_hit_rate"])
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD |
| `analytics.cache_hits` | integer | Number of cache hits |
| `analytics.cache_hit_rate` | number | Cache hit rate (0-1) |
| `analytics.avg_latency_ms` | integer | Average response latency in milliseconds |
| `analytics.success_rate` | number | Success rate (0-1) |
| `analytics.error_count` | integer | Number of failed requests |
| `analytics.endpoint_distribution` | object | Request count per endpoint |
| `analytics.provider_distribution` | object | Request count per AI provider |
| `analytics.period` | object | The time period queried |
| `by_endpoint` | array | Breakdown by endpoint with requests, cost, and latency |

### Grouping options

The `group_by` parameter changes the breakdown key in the response:

| Value | Breakdown key | Description |
|-------|---------------|-------------|
| `endpoint` | `by_endpoint` | Group by API endpoint (default) |
| `day` | `by_day` | Group by calendar day |
| `provider` | `by_provider` | Group by AI provider |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
