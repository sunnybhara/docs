Retrieve aggregated analytics across your API usage. Includes request counts, costs, cache hit rates, latency, success rates, and breakdowns by endpoint, day, or provider. This endpoint is **free**.

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601), e.g. `2025-01-01T00:00:00Z` |
| `to` | string | No | End date (ISO 8601), e.g. `2025-01-31T23:59:59Z` |
| `group_by` | string | No | Group results by: `endpoint`, `day`, `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2025-01-01T00:00:00Z&to=2025-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.analytics({
  from: '2025-01-01T00:00:00Z',
  to: '2025-01-31T23:59:59Z',
  groupBy: 'endpoint'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.analytics(
    from_date="2025-01-01T00:00:00Z",
    to_date="2025-01-31T23:59:59Z",
    group_by="endpoint"
)
```

</CodeGroup>

### Group by `day` example

```bash
curl "https://api.trynawa.com/v1/analytics?group_by=day&from=2025-01-01" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

Returns `by_day` with date-keyed entries:

```json
{
  "by_day": [
    {"day": "2025-01-15", "requests": 145, "cost": 0.87, "avg_latency_ms": 810},
    {"day": "2025-01-14", "requests": 132, "cost": 0.792, "avg_latency_ms": 790}
  ]
}
```

### Group by `provider` example

```bash
curl "https://api.trynawa.com/v1/analytics?group_by=provider" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

Returns `by_provider` with provider-keyed entries:

```json
{
  "by_provider": [
    {"provider": "allam", "requests": 2800, "cost": 16.8, "avg_latency_ms": 850},
    {"provider": "claude", "requests": 1721, "cost": 10.326, "avg_latency_ms": 720}
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD |
| `analytics.cache_hits` | integer | Number of requests served from cache |
| `analytics.cache_hit_rate` | number | Cache hit ratio (0-1) |
| `analytics.avg_latency_ms` | integer | Average response latency in ms |
| `analytics.success_rate` | number | Success ratio (0-1) |
| `analytics.error_count` | integer | Total error responses |
| `analytics.endpoint_distribution` | object | Request count by endpoint |
| `analytics.provider_distribution` | object | Request count by provider |
| `analytics.period` | object | The `from` and `to` dates of the query |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid `group_by` value |
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
