Retrieve aggregated analytics on API usage, performance, and provider distribution.

<Note>
  This endpoint is **free** -- 0 credits, no cost.
</Note>

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601). |
| `to` | string | No | End date (ISO 8601). |
| `group_by` | string | No | Group results by: `endpoint`, `day`, `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2025-01-01T00:00:00Z&to=2025-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
const { data, error } = await nawa.analytics.get({
  from: '2025-01-01T00:00:00Z',
  to: '2025-01-31T23:59:59Z',
  groupBy: 'endpoint'
})
```

```python Python
result = nawa.analytics.get(
    from_date="2025-01-01T00:00:00Z",
    to_date="2025-01-31T23:59:59Z",
    group_by="endpoint",
)
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD |
| `analytics.cache_hits` | integer | Number of semantic cache hits |
| `analytics.cache_hit_rate` | number | Cache hit ratio (0-1) |
| `analytics.avg_latency_ms` | number | Average response time in milliseconds |
| `analytics.success_rate` | number | Success ratio (0-1) |
| `analytics.error_count` | integer | Total error count |
| `analytics.period` | object | Time range with `from` and `to` (ISO 8601) |
| `by_endpoint` | object[] | Breakdown by endpoint with `endpoint`, `requests`, `cost`, `avg_latency_ms` |

<Tip>
  Use `group_by=provider` to see usage split across AI providers (allam, claude, gemini). Use `group_by=day` for daily trends.
</Tip>
