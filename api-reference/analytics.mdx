Retrieve aggregated analytics across your API usage including request counts, cost breakdown, cache hit rates, and provider distribution. This endpoint is **free**.

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601) |
| `to` | string | No | End date (ISO 8601) |
| `group_by` | string | No | Group results by: `endpoint`, `day`, `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2025-01-01T00:00:00Z&to=2025-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.analytics({
  from: '2025-01-01T00:00:00Z',
  to: '2025-01-31T23:59:59Z',
  groupBy: 'endpoint'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="nawa_live_sk_xxx")

result = nawa.analytics(
    from_date="2025-01-01T00:00:00Z",
    to_date="2025-01-31T23:59:59Z",
    group_by="endpoint"
)
```

</CodeGroup>

### Group by options

The breakdown array key changes based on the `group_by` parameter:

| `group_by` | Array key | Entry key |
|------------|-----------|-----------|
| `endpoint` | `by_endpoint` | `endpoint` (e.g. `/v1/classify`) |
| `day` | `by_day` | `date` (e.g. `2025-01-15`) |
| `provider` | `by_provider` | `provider` (e.g. `allam`) |

Each breakdown entry includes: `requests`, `cost`, and `avg_latency_ms`.

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD |
| `analytics.cache_hits` | integer | Number of cache hits |
| `analytics.cache_hit_rate` | number | Cache hit rate (0-1) |
| `analytics.avg_latency_ms` | integer | Average response latency in ms |
| `analytics.success_rate` | number | Percentage of successful requests (0-1) |
| `analytics.error_count` | integer | Number of error responses |
| `analytics.endpoint_distribution` | object | Request count per endpoint |
| `analytics.provider_distribution` | object | Request count per provider |
| `analytics.period` | object | Date range (`from`, `to`) |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
