Retrieve aggregated analytics across your API usage. Includes request counts, costs, cache hit rates, latency, and breakdowns by endpoint, day, or provider. This endpoint is **free**.

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601), e.g., `2026-01-01T00:00:00Z`. |
| `to` | string | No | End date (ISO 8601), e.g., `2026-01-31T23:59:59Z`. |
| `group_by` | string | No | Group results by: `endpoint`, `day`, or `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2026-01-01T00:00:00Z&to=2026-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.analytics({
  from: '2026-01-01T00:00:00Z',
  to: '2026-01-31T23:59:59Z',
  groupBy: 'endpoint'
})

console.log(result.analytics.total_requests)
console.log(result.by_endpoint)
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="nawa_live_sk_xxx")

result = nawa.analytics(
    from_date="2026-01-01T00:00:00Z",
    to_date="2026-01-31T23:59:59Z",
    group_by="endpoint",
)

print(result.analytics.total_requests)
print(result.by_endpoint)
```

</CodeGroup>

<Note>
  The breakdown key changes based on `group_by`: `by_endpoint`, `by_day`, or `by_provider`. When grouping by `day`, each item includes a `day` field with the date (YYYY-MM-DD). When grouping by `provider`, each item includes a `provider` field.
</Note>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD (rounded to 4 decimal places) |
| `analytics.cache_hits` | integer | Number of requests served from semantic cache |
| `analytics.cache_hit_rate` | number | Cache hit ratio (0 to 1, rounded to 2 decimal places) |
| `analytics.avg_latency_ms` | number | Average latency in milliseconds |
| `analytics.success_rate` | number | Success ratio (0 to 1, rounded to 2 decimal places) |
| `analytics.error_count` | integer | Total failed requests |
| `analytics.endpoint_distribution` | object | Request counts by endpoint path |
| `analytics.provider_distribution` | object | Request counts by AI provider |
| `analytics.period` | object | The `from` and `to` date range (null if not specified) |
| `by_endpoint` / `by_day` / `by_provider` | array | Breakdown items sorted by request count descending |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
