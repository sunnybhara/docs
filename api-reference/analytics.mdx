Retrieve aggregated analytics across your API usage including request counts, costs, cache hit rates, latency, and success rates. This endpoint is **free**.

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601), e.g. `2026-01-01T00:00:00Z` |
| `to` | string | No | End date (ISO 8601), e.g. `2026-01-31T23:59:59Z` |
| `group_by` | string | No | Group breakdown by: `endpoint`, `day`, `provider`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/analytics?from=2026-01-01T00:00:00Z&to=2026-01-31T23:59:59Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const response = await fetch('https://api.trynawa.com/v1/analytics?group_by=endpoint', {
  headers: { 'Authorization': `Bearer ${process.env.NAWA_API_KEY}` }
})
const { result } = await response.json()
```

```python Python
import requests

response = requests.get(
    "https://api.trynawa.com/v1/analytics",
    params={"group_by": "endpoint"},
    headers={"Authorization": "Bearer your_api_key"}
)
data = response.json()
```

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `analytics.total_requests` | integer | Total API requests in the period |
| `analytics.total_cost` | number | Total cost in USD |
| `analytics.cache_hits` | integer | Number of cache hits |
| `analytics.cache_hit_rate` | number | Ratio of cache hits to total requests (0-1) |
| `analytics.avg_latency_ms` | integer | Average response time in milliseconds |
| `analytics.success_rate` | number | Ratio of successful requests (0-1) |
| `analytics.error_count` | integer | Number of requests that returned errors |
| `analytics.endpoint_distribution` | object | Request counts by endpoint |
| `analytics.provider_distribution` | object | Request counts by AI provider |
| `analytics.period` | object | The queried time range |
| `by_endpoint` / `by_day` / `by_provider` | array | Breakdown grouped by the `group_by` parameter |

<Info>
  The breakdown array key name changes based on `group_by`: `by_endpoint`, `by_day`, or `by_provider`.
</Info>

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid `group_by` value |
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
