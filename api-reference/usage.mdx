Retrieve your API usage statistics including request counts, credit consumption, and cache hit rates. This endpoint is **free** -- 0 credits, no cost.

## Request

### Query parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `from` | string | No | -- | Start date (ISO 8601) |
| `to` | string | No | -- | End date (ISO 8601) |
| `group_by` | string | No | `endpoint` | Group by: `endpoint` or `day` |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2025-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.usage({
  from: '2025-01-01T00:00:00Z',
  groupBy: 'endpoint'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.usage(
    from_date="2025-01-01T00:00:00Z",
    group_by="endpoint"
)
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total credit cost in USD |
| `usage.cache_hits` | integer | Total cache hits |
| `usage.cache_hit_rate` | number | Cache hit ratio (0-1) |
| `usage.period` | object | The queried time period with `from` and `to` |
| `by_endpoint` | array | Breakdown by endpoint (or `by_day` when `group_by=day`) |

Each breakdown entry contains:
- The grouping key (`endpoint` or `date`)
- `requests` -- number of requests
- `cost` -- total cost in USD
- `cache_hits` -- number of cache hits

<Note>
  When using `group_by=day`, the breakdown key changes to `by_day` and each entry has a `date` field instead of `endpoint`.
</Note>

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid `group_by` value |
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
