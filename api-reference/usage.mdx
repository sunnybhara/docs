Retrieve your API usage statistics including request counts, credit consumption, and cache hit rates. This endpoint is **free**.

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601). Default: no lower bound. |
| `to` | string | No | End date (ISO 8601). Default: no upper bound. |
| `group_by` | string | No | Group by: `endpoint`, `day`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2025-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.usage({
  from: '2025-01-01T00:00:00Z',
  groupBy: 'endpoint'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.usage(
    from_date="2025-01-01T00:00:00Z",
    group_by="endpoint"
)
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total cost in USD |
| `usage.cache_hits` | integer | Number of cache hit responses |
| `usage.cache_hit_rate` | number | Cache hit percentage (0-1) |
| `usage.period` | object | The queried time range |
| `by_endpoint` | object[] | Breakdown per endpoint with `endpoint`, `requests`, `cost`, `cache_hits` |

<Tip>
  Use the `group_by=day` parameter to get a daily breakdown instead of per-endpoint. The response key changes to `by_day` accordingly.
</Tip>

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
