Retrieve your API usage statistics including request counts, cost, and cache hit rates. This endpoint is **free**.

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601). |
| `to` | string | No | End date (ISO 8601). |
| `group_by` | string | No | Group by: `endpoint` or `day`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2025-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
const params = new URLSearchParams({
  from: '2025-01-01T00:00:00Z',
  group_by: 'endpoint'
})

const response = await fetch(`https://api.trynawa.com/v1/usage?${params}`, {
  headers: {
    'Authorization': 'Bearer nawa_test_sk_xxx'
  }
})

const data = await response.json()
console.log(data.result.usage.total_requests)
console.log(data.result.usage.total_cost)
```

```python Python
import requests

response = requests.get(
    "https://api.trynawa.com/v1/usage",
    headers={"Authorization": "Bearer nawa_test_sk_xxx"},
    params={
        "from": "2025-01-01T00:00:00Z",
        "group_by": "endpoint"
    }
)

data = response.json()
print(data["result"]["usage"]["total_requests"])
print(data["result"]["usage"]["total_cost"])
```

</CodeGroup>

## Response

### Success response (200)

```json
{
  "result": {
    "usage": {
      "total_requests": 150,
      "total_cost": 0.84,
      "cache_hits": 30,
      "cache_hit_rate": 0.20,
      "period": {
        "from": "2025-01-01T00:00:00Z",
        "to": null
      }
    },
    "by_endpoint": [
      {
        "endpoint": "/v1/classify",
        "requests": 100,
        "cost": 0.60,
        "cache_hits": 20
      },
      {
        "endpoint": "/v1/comments/reply",
        "requests": 50,
        "cost": 0.24,
        "cache_hits": 10
      }
    ]
  },
  "success": true,
  "errors": [],
  "request_id": "req_nw_a1b2c3d4e5f67890abcdef12"
}
```

<Note>
  When `group_by=day`, the breakdown key becomes `by_day` and each item includes a `date` field (`YYYY-MM-DD`) instead of `endpoint`.
</Note>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total cost in USD |
| `usage.cache_hits` | integer | Number of semantic cache hits |
| `usage.cache_hit_rate` | number | Cache hit percentage (0-1) |
| `usage.period` | object | The time range queried (`from` and `to`) |
| `by_endpoint` | object[] | Per-endpoint breakdown with `requests`, `cost`, `cache_hits` |

### Error responses

| Status | Type | When |
|--------|------|------|
| 400 | `invalid_request_error` | Invalid `group_by` value (must be `endpoint` or `day`) |
| 401 | `authentication_error` | Invalid or missing API key |
| 405 | `invalid_request_error` | Non-GET request method |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
