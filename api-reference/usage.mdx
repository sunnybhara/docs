Retrieve your API usage statistics including request counts, credit consumption, and cache hit rates. This endpoint is **free**.

## Request

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601) |
| `to` | string | No | End date (ISO 8601) |
| `group_by` | string | No | Group by: `endpoint` or `day`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2026-03-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const { data, error } = await nawa.usage.get({
  from: '2026-03-01T00:00:00Z',
  groupBy: 'endpoint'
})
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.usage.get(
    from_date="2026-03-01T00:00:00Z",
    group_by="endpoint"
)
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total credit cost in USD |
| `usage.cache_hits` | integer | Number of cache hits |
| `usage.cache_hit_rate` | number | Cache hit ratio (0-1) |
| `usage.period` | object | The queried date range |
| `by_endpoint` | array | Breakdown per endpoint with `requests`, `cost`, and `cache_hits` |

When using `group_by=day`, the breakdown array uses `by_day` and each entry has a `date` field instead of `endpoint`:

```json
{
  "by_day": [
    { "date": "2026-03-01", "requests": 420, "cost": 0.95, "cache_hits": 98 },
    { "date": "2026-03-02", "requests": 380, "cost": 0.86, "cache_hits": 85 }
  ]
}
```
