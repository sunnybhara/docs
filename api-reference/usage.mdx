Retrieve your API usage statistics including request counts, credit consumption, and cache hit rates. This endpoint is **free**.

## Request

### Query parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `from` | string | No | -- | Start date (ISO 8601) |
| `to` | string | No | -- | End date (ISO 8601) |
| `group_by` | string | No | `endpoint` | Group results by: `endpoint` or `day` |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2025-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_test_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.usage.get({
  from: '2025-01-01T00:00:00Z',
  groupBy: 'endpoint'
})

console.log(result.data.usage.total_requests)
console.log(result.data.usage.total_cost)
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="your_api_key")

result = nawa.usage.get(
    from_date="2025-01-01T00:00:00Z",
    group_by="endpoint"
)

print(result.data.usage.total_requests)
print(result.data.usage.total_cost)
```

</CodeGroup>

<Note>
  When `group_by=day`, the breakdown key is `by_day` with `date`, `requests`, `cost`, and `cache_hits` fields. Breakdown is sorted by request count (highest first).
</Note>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total cost in USD |
| `usage.cache_hits` | integer | Number of cache hits |
| `usage.cache_hit_rate` | number | Cache hit rate 0-1 |
| `usage.period` | object | Time period with `from` and `to` |
| `by_endpoint` | array | Breakdown by endpoint (when `group_by=endpoint`) |
| `by_day` | array | Breakdown by day (when `group_by=day`) |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
