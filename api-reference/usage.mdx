Retrieve your API usage statistics including request counts, costs, and cache hit rates. This endpoint is **free**.

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601), e.g., `2026-01-01T00:00:00Z`. |
| `to` | string | No | End date (ISO 8601), e.g., `2026-01-31T23:59:59Z`. |
| `group_by` | string | No | Group by: `endpoint` or `day`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2026-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY })

const result = await nawa.usage({
  from: '2026-01-01T00:00:00Z',
  groupBy: 'endpoint'
})

console.log(result.usage.total_requests)
console.log(result.by_endpoint)
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="nawa_live_sk_xxx")

result = nawa.usage(
    from_date="2026-01-01T00:00:00Z",
    group_by="endpoint",
)

print(result.usage.total_requests)
print(result.by_endpoint)
```

</CodeGroup>

<Note>
  The breakdown key changes based on `group_by`: `by_endpoint` or `by_day`. When grouping by `day`, each item uses a `date` field (YYYY-MM-DD) instead of `endpoint`.
</Note>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total cost in USD (rounded to 4 decimal places) |
| `usage.cache_hits` | integer | Number of requests served from semantic cache |
| `usage.cache_hit_rate` | number | Cache hit ratio (0 to 1, rounded to 2 decimal places) |
| `usage.period` | object | The `from` and `to` date range (null if not specified) |
| `by_endpoint` / `by_day` | array | Breakdown items sorted by request count descending |

### Breakdown item fields

**When `group_by=endpoint`:**

| Field | Type | Description |
|-------|------|-------------|
| `endpoint` | string | API endpoint path |
| `requests` | integer | Request count |
| `cost` | number | Cost in USD |
| `cache_hits` | integer | Cache hit count |

**When `group_by=day`:**

| Field | Type | Description |
|-------|------|-------------|
| `date` | string | Date (YYYY-MM-DD) |
| `requests` | integer | Request count |
| `cost` | number | Cost in USD |
| `cache_hits` | integer | Cache hit count |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
