Retrieve your API usage statistics including request counts, credit consumption, and cache hit rates.

<Note>
  This endpoint is **free** -- 0 credits, no cost.
</Note>

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer nawa_live_sk_xxx` or `Bearer nawa_test_sk_xxx` |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (ISO 8601). Default: start of current month. |
| `to` | string | No | End date (ISO 8601). Default: now. |
| `group_by` | string | No | Group results by: `endpoint`, `day`. Default: `endpoint`. |

### Example request

<CodeGroup>

```bash cURL
curl "https://api.trynawa.com/v1/usage?from=2025-01-01T00:00:00Z&group_by=endpoint" \
  -H "Authorization: Bearer nawa_live_sk_xxx"
```

```typescript TypeScript
import { Nawa } from "@nawalabs/sdk";

const nawa = new Nawa({ apiKey: process.env.NAWA_API_KEY });

const result = await nawa.usage.get({
  from: "2025-01-01T00:00:00Z",
  groupBy: "endpoint",
});

console.log(result.usage.total_requests);
console.log(result.usage.cache_hit_rate);
```

```python Python
from nawa import Nawa

nawa = Nawa(api_key="nawa_live_sk_xxx")

result = nawa.usage.get(
    from_date="2025-01-01T00:00:00Z",
    group_by="endpoint",
)

print(result.usage["total_requests"])
print(result.usage["cache_hit_rate"])
```

</CodeGroup>

### Result fields

| Field | Type | Description |
|-------|------|-------------|
| `usage.total_requests` | integer | Total API requests in the period |
| `usage.total_cost` | number | Total cost in USD |
| `usage.cache_hits` | integer | Number of requests served from cache |
| `usage.cache_hit_rate` | number | Cache hit rate (0-1) |
| `usage.period` | object | The queried time period |
| `by_endpoint` | array | Breakdown by endpoint (when `group_by=endpoint`) |
| `by_day` | array | Breakdown by day (when `group_by=day`) |

### Error responses

| Status | Type | When |
|--------|------|------|
| 401 | `authentication_error` | Invalid or missing API key |
| 429 | `rate_limit_error` | Rate limit exceeded |
| 500 | `api_error` | Internal error |
