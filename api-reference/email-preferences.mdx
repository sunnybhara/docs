Manage email preferences without requiring a user login. Every NAWA transactional email includes an unsubscribe link with a signed JWT token. This endpoint verifies that token and lets users update their email category preferences.

<Note>
  This is a Supabase Edge Function endpoint. The base URL differs from the main NAWA API. Authentication uses a signed JWT token (included in unsubscribe links), not your API key.
</Note>

## Base URL

```
https://<your-project-ref>.supabase.co/functions/v1/email-preferences-api
```

Replace `<your-project-ref>` with your Supabase project reference.

## Authentication

This endpoint uses **token-based auth** instead of API keys. The `token` is a signed JWT included in every NAWA email unsubscribe link. Tokens are valid for 30 days.

No `Authorization` header is required. Pass the token in the request body.

## Email categories

NAWA groups emails into four categories. Each maps to one or more preference columns:

| Category | Label | Description | Can disable? |
|----------|-------|-------------|-------------|
| `product` | Product updates | Feature announcements, upgrade prompts | Yes |
| `intelligence` | Intelligence reports | Weekly summaries, activity digests, urgent alerts | Yes |
| `billing` | Billing and account | Payment notifications, dunning emails, password resets | No |
| `onboarding` | Tips and onboarding | Welcome emails, trial reminders, activation nudges | Yes |

<Warning>
  Billing emails cannot be disabled. They are required for account integrity and payment compliance.
</Warning>

## Actions

All requests use `POST` with a JSON body containing an `action` field.

### Verify and unsubscribe

Verify a token and automatically disable the email category that triggered the unsubscribe. Use this when handling one-click unsubscribe links.

#### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | string | Yes | `"verify_and_unsub"` |
| `token` | string | Yes | Signed JWT from the unsubscribe link |
| `flow_id` | string | No | The email flow ID (e.g., `F10_weekly_intelligence`). Determines which category to disable. |

#### Example request

<CodeGroup>

```bash cURL
curl -X POST https://<your-project-ref>.supabase.co/functions/v1/email-preferences-api \
  -H "Content-Type: application/json" \
  -d '{
    "action": "verify_and_unsub",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "flow_id": "F10_weekly_intelligence"
  }'
```

```python Python
import requests

response = requests.post(
    "https://<your-project-ref>.supabase.co/functions/v1/email-preferences-api",
    json={
        "action": "verify_and_unsub",
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "flow_id": "F10_weekly_intelligence",
    },
)
data = response.json()
```

```typescript TypeScript
const response = await fetch(
  'https://<your-project-ref>.supabase.co/functions/v1/email-preferences-api',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      action: 'verify_and_unsub',
      token: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
      flow_id: 'F10_weekly_intelligence',
    }),
  }
)
const data = await response.json()
```

</CodeGroup>

#### Success response (200)

```json

  "ok": true,
  "unsubscribed_category": "Intelligence reports",
  "categories": {
    "product": true,
    "intelligence": false,
    "billing": true,
    "onboarding": true,
    "monthly_summary": false,
    "all_emails": true
  }
}
```

### Update preferences

Update one or more category toggles. Pass only the categories you want to change.

#### Body parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | string | Yes | `"update"` |
| `token` | string | Yes | Signed JWT from the email link |
| `prefs` | object | Yes | Category toggles to update |

The `prefs` object accepts these keys:

| Key | Type | Description |
|-----|------|-------------|
| `product` | boolean | Enable or disable product update emails |
| `intelligence` | boolean | Enable or disable intelligence report emails |
| `onboarding` | boolean | Enable or disable onboarding emails |
| `monthly_summary` | boolean | Enable or disable the monthly summary (win-back digest) |

<Tip>
  The `monthly_summary` option is a lightweight digest for users who turn off other categories. Consider enabling it as an alternative to full unsubscribe.
</Tip>

#### Example request

<CodeGroup>

```bash cURL
curl -X POST https://<your-project-ref>.supabase.co/functions/v1/email-preferences-api \
  -H "Content-Type: application/json" \
  -d '{
    "action": "update",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "prefs": {
      "intelligence": false,
      "monthly_summary": true
    }
  }'
```

## Error responses

| Status | Body | Cause |
|--------|------|-------|
| 400 | `{"error": "Token required"}` | Missing `token` in the request body |
| 400 | `{"error": "prefs object required"}` | `update` action called without a `prefs` object |
| 400 | `{"error": "Unknown action: <action>"}` | Invalid `action` value |
| 401 | `{"error": "Invalid or expired token"}` | Token signature is invalid, token is expired (older than 30 days), or token issuer does not match |
| 405 | `{"error": "POST only"}` | Request used a method other than POST |
| 500 | `{"error": "Internal error"}` | Server-side failure |

## Token structure

Unsubscribe tokens are HMAC-SHA256 signed JWTs with the following claims:

| Claim | Value |
|-------|-------|
| `sub` | User ID |
| `iss` | `nawa-email` |
| `iat` | Issued-at timestamp (Unix seconds) |
| `exp` | Expiration (30 days from issue) |

Tokens are generated automatically when NAWA sends an email. You do not need to create tokens manually.
