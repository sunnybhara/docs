The official NAWA TypeScript SDK provides a type-safe client with `{ data, error }` returns, auto-pagination, and built-in retry logic.

## Installation

<CodeGroup>

```bash npm
npm install @nawalabs/sdk
```

```bash pnpm
pnpm add @nawalabs/sdk
```

```bash yarn
yarn add @nawalabs/sdk
```

```bash bun
bun add @nawalabs/sdk
```

</CodeGroup>

## Quick start

```typescript
import { Nawa } from '@nawalabs/sdk'

const nawa = new Nawa({
  apiKey: process.env.NAWA_API_KEY  // nawa_live_sk_xxx or nawa_test_sk_xxx
})

// Classify Arabic text
const { data, error } = await nawa.classify({
  text: 'ما شاء الله عليك، محتوى رهيب!'
})

if (error) {
  console.error(error.type, error.message)
} else {
  console.log(data.intent)    // ["praise"]
  console.log(data.dialect)   // "gulf"
  console.log(data.sentiment) // "positive"
}
```

<Note>
  The SDK uses `{ data, error }` returns instead of throwing exceptions. This makes error handling explicit and avoids uncaught promise rejections.
</Note>

## Classify text

```typescript
const { data, error } = await nawa.classify({
  text: 'متى الجزء الثاني؟'
})

if (data) {
  console.log(data.intent)            // ["question"]
  console.log(data.dialect)           // "gulf"
  console.log(data.dialect_confidence) // 0.95
  console.log(data.provider)          // "allam"
}
```

## Rubric classification

```typescript
const { data, error } = await nawa.rubric.classify({
  text: 'وين الترجمة العربية؟',
  rubric: {
    categories: [
      { name: 'translation_request', description: 'User wants subtitles or translation' },
      { name: 'technical_issue', description: 'Audio/video problems' },
      { name: 'content_feedback', description: 'Feedback on content quality' }
    ],
    multi_label: false,
    confidence_threshold: 0.5
  }
})

if (data) {
  console.log(data.categories) // [{ name: "translation_request", confidence: 0.94 }]
}
```

## Classify and reply

```typescript
const { data, error } = await nawa.comments.reply({

const isValid = Nawa.webhooks.verify({

```

## Error handling

The SDK provides typed errors for each error category:

```typescript
import { Nawa, NawaAuthError, NawaRateLimitError, NawaCreditsError } from '@nawalabs/sdk'

const { data, error } = await nawa.classify({ text: '...' })

if (error) {
  if (error instanceof NawaAuthError) {
    // Invalid, expired, or revoked API key
    console.error('Auth error:', error.code) // "invalid_api_key"
  } else if (error instanceof NawaRateLimitError) {
    // Rate limit exceeded
    console.log(`Retry after ${error.retryAfter}ms`)
  } else if (error instanceof NawaCreditsError) {
    // Insufficient credits
    console.error('Buy credits:', error.suggestedAction)
  } else {
    // Server error -- safe to retry
    console.error('API error:', error.message)
  }
}
```

## Configuration

```typescript
const nawa = new Nawa({
  apiKey: process.env.NAWA_API_KEY,

  // Retry configuration
  maxRetries: 3,          // Default: 3
  retryDelay: 1000,       // Base delay in ms. Default: 1000
  retryBackoff: 'exponential', // "exponential" | "linear" | "none"

  // Timeout
  timeout: 30_000,        // Request timeout in ms. Default: 30000

  // Base URL override (for testing)
  baseUrl: 'https://api.trynawa.com' // Default
})
```
