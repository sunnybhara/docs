NAWA uses a prepaid credit system. Buy credit packs, and each API call deducts from your balance.

## Credit packs

| Pack | Price | Best for |
|------|-------|----------|
| **Starter** | $25 | Prototyping and small projects |
| **Builder** | $50 | Growing applications |
| **Scale** | $100 | Production workloads |
| **Enterprise** | $500 | High-volume usage |

Purchase credit packs at [trynawa.com/developers/keys](https://trynawa.com/developers/keys).

## Per-endpoint pricing

| Endpoint | Cost per call | Credits | Notes |
|----------|--------------|---------|-------|
| `POST /v1/classify` | $0.006 | 6 | Full classification with dialect detection |
| `POST /v1/rubric-classify` | $0.003 | 3 | Custom rubric classification |
| `POST /v1/comments/reply` | $0.008 | 8 | Classification + AI reply |
| `POST /v1/feedback` | **Free** | 0 | RLHF feedback submission |
| `GET /v1/comments` | **Free** | 0 | List classified comments |
| `GET /v1/analytics` | **Free** | 0 | Analytics data |
| `GET /v1/health` | **Free** | 0 | Health check (no auth required) |
| `GET /v1/usage` | **Free** | 0 | Usage statistics |

<Tip>
  Semantic cache hits (`X-NAWA-Cache: HIT`) cost **$0** and use 0 credits, even on paid endpoints. Classifying the same or semantically similar text is free after the first call.
</Tip>

## Free tier

Every account gets **500 free requests/month** on `/v1/rubric-classify`, no credit card required. This resets on the 1st of each month.

Free API keys also get **100 lifetime requests** across all endpoints with a 14-day expiry.

## Balance and spend tracking

The key listing endpoint returns your current balance and lifetime spend:

```json
{
  "billing": {
    "credit_balance": 42.50,
    "lifetime_spend": 57.50,
    "sandbox_requests_used": 23,
    "sandbox_requests_limit": 100
  }
}
```

| Field | Description |
|-------|-------------|
| `credit_balance` | Current credit balance in USD |
| `lifetime_spend` | Total amount spent across all time |
| `sandbox_requests_used` | Combined sandbox requests across active free keys |
| `sandbox_requests_limit` | Maximum sandbox requests (100) |

## Balance headers

Every response includes balance information:

| Header | Description | Example |
|--------|-------------|---------||
| `X-NAWA-Balance` | Current credit balance in USD | `12.45` |
| `X-NAWA-Balance-Warning` | Warning when balance is low | `low_balance` |

The `X-NAWA-Balance-Warning` header appears when your balance drops below $5.

## Grace period

When your balance hits **$0**, NAWA provides a grace period of the lesser of:

- **1 hour** of continued access, or
- **$2 worth** of API calls

This prevents your production traffic from immediately failing while you add credits.

<Warning>
  The grace period is a one-time buffer. Once it expires, all paid endpoints return `402 insufficient_credits` until you add credits.
</Warning>

## Auto top-up

Enable auto top-up to never run out of credits:

| Setting | Default |
|---------|---------||
| **Trigger threshold** | $5.00 |
| **Top-up amount** | $25.00 |
| **Monthly cap** | $100.00 |
| **Cooldown** | 24 hours between top-ups |
| **Anomaly detection** | Enabled. Pauses top-up on 5x spend spikes. |

Configure auto top-up at [trynawa.com/developers/keys](https://trynawa.com/developers/keys).

<Info>
  Anomaly detection monitors your spending pattern. If your spend rate suddenly increases 5x above your trailing 7-day average, auto top-up pauses and you receive an email alert. This protects against runaway costs from misconfigured integrations.
</Info>
